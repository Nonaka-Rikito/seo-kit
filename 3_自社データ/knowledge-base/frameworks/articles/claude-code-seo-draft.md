# Claude CodeでSEO分析基盤をゼロから構築する全手順【実務者向け】

SEOの業務フローには、キーワード調査、競合分析、記事の構成・執筆、公開後の効果測定と、複数のツールをまたぐ作業が常について回ります。Ahrefs、Google Search Console、Google Analytics、スプレッドシート、WordPress——それぞれにログインし、データを転記し、分析を重ねるこの一連の作業に、毎月かなりの時間を使っているSEO担当の方は少なくないはずです。

しかし、これらの断片化した作業を一つのインターフェースに集約できるとしたらどうでしょうか。Claude Codeは、外部ツールとの接続・ファイル操作・コード実行を備えたAIエージェントです。単に「記事を書かせる」だけでなく、Ahrefsからキーワードデータを取得し、GSCの順位変動を分析し、その結果に基づいて記事を制作する——この「分析→判断→実行」のサイクルを、一つのターミナル上で完結させることができます。

本記事では、Claude Codeを使ったSEO分析基盤を文字どおりゼロから構築する手順を解説します。環境構築からMCP接続、スキル設計、実際のワークフロー運用までの全工程を、コンサルティング現場での実践経験をもとにまとめました。SEOの基礎知識はあるがClaude Codeは初めてという方を主な読者として想定しています。

## Claude CodeがSEO業務にもたらす変化

Claude Codeは、AIとの対話をファイルシステムや外部APIと直結させたエージェント型のツールです。SEO業務の文脈では、チャットAIにできなかった「実行」の部分を担えるようになった点が最大の転換点になります。

### チャットAIとClaude Codeの決定的な違い

ChatGPTやClaude（チャット版）に「このキーワードの検索ボリュームを調べて」と頼んでも、返ってくるのは推測値か一般的な知識です。実際のAhrefsのデータにアクセスすることはできません。

Claude Codeはここが根本的に異なります。MCP（Model Context Protocol）という接続規格を通じて、Ahrefs、Google Search Console、Slackなどの外部ツールに直接アクセスできます。つまり、AIが推測ではなく実データをもとに分析し、その結果をファイルに書き出し、必要であればコードを実行して可視化するところまでを一気に処理します。

チャットAIが「助言をくれるアドバイザー」だとすれば、Claude Codeは「データを引っ張ってきて分析し、レポートにまとめ、記事の下書きまで作る実行力のあるSEO担当者」に近い存在です。

### SEO業務で特に効くポイント

Claude CodeがSEO業務にフィットする理由は、大きく3つあります。

1つ目は**データ分析の直接実行**です。AhrefsのAPIからキーワードデータや被リンク情報を取得し、GSCのクリック数・表示回数と突き合わせる——この作業をターミナル上の対話だけで完結させることができます。スプレッドシートに転記して関数を組む手間がなくなります。

2つ目は**記事制作の一気通貫化**です。キーワード調査→構成案作成→本文執筆→リライトという4工程を、同じセッション内で連続実行できます。各工程の出力が次の工程の入力として自動的に引き継がれるため、コピー&ペーストで情報を移す作業がなくなります。

3つ目は**テクニカルSEOの自動化**です。サイト全体のmeta情報チェック、構造化データの検証、内部リンクの分析といった技術的な監査を、Claude Codeのファイル操作能力とWeb取得機能を組み合わせて実行できます。

## SEO分析基盤の全体設計

Claude CodeでSEO基盤を構築する際、単にツールをインストールするだけでは機能しません。3つの構成要素を理解し、それぞれの役割に沿って設計する必要があります。

### 構成要素と役割分担

SEO分析基盤は、次の3層で構成されます。

**MCP（Model Context Protocol）** は、Claude Codeと外部ツールをつなぐ接続層です。Ahrefs、GSC、Slack、Notionなど、SEO業務に必要なデータソースをClaude Codeから直接操作できるようにします。MCPは「Claude Codeの手足」にあたる部分で、ここが充実するほどAIが扱えるデータの幅が広がります。

**スキル** は、繰り返し実行するSEOワークフローをMarkdownで定義したテンプレートです。「キーワード調査はこの手順で、この形式で出力する」といったルールをスキルとして定義しておけば、スラッシュコマンド一発で同じ品質のアウトプットを再現できます。「Claude Codeの仕事の型」と考えるとわかりやすいでしょう。

**CLAUDE.md** は、プロジェクト固有の知識やルールを定義するファイルです。担当クライアントの情報、分析時の判断基準、出力フォーマットのルールなどをここに書いておくと、Claude Codeがセッション開始時に自動で読み込み、文脈を踏まえた応答を返すようになります。いわば「Claude Codeの業務マニュアル」です。

この3層は独立ではなく、相互に連携して機能します。たとえば、スキル「/seo-recovery-plan」を実行すると、MCP経由でAhrefsのAPIからキーワードデータを取得し、CLAUDE.mdに定義されたクライアントの目標値を参照しながら、リカバリープランを生成する——という流れになります。

### 最小構成と拡張パターン

いきなりフル構成を目指す必要はありません。段階的に拡張する方が、各層の役割を理解しながら進められます。

**Step 1: 最小構成（CLAUDE.md + WebSearch）**

Claude Codeには標準でWeb検索機能が搭載されています。CLAUDE.mdにクライアント情報と分析ルールを書いておくだけでも、キーワードの検索意図分析、上位記事の内容調査、記事構成案の作成といった作業は十分に回せます。まずはここから始めて、Claude Codeの動きに慣れるのがおすすめです。

**Step 2: Ahrefs MCPの追加**

Web検索だけでは取得できない定量データ——検索ボリューム、キーワード難易度（KD）、被リンク数、ドメインレーティングなど——をAhrefs MCPで補完します。この段階で、データに裏付けられたキーワード選定や競合分析が可能になります。

**Step 3: GSC・Slack・Notionの追加**

GSCを追加すれば自社サイトの実績データにもとづく効果測定ができ、Slackを追加すればクライアントとのコミュニケーション文脈をAIが把握できるようになります。Notionとの接続で、プロジェクト管理やナレッジベースとの連携も実現します。ここまで来れば、SEO業務のほぼ全工程をClaude Code上で回せる基盤が完成します。

## 環境構築の実践手順

ここからは具体的な構築手順に入ります。Claude Codeの導入からMCP接続まで、順を追って解説します。

### Claude Codeのインストールと初期設定

Claude Codeの導入にはNode.js環境が必要です。以下のコマンドで一気にセットアップできます。

```bash
# Node.jsがインストール済みであることを確認
node -v

# Claude Codeをグローバルインストール
npm install -g @anthropic-ai/claude-code

# プロジェクトフォルダを作成して移動
mkdir seo-workspace && cd seo-workspace

# Claude Codeを起動
claude
```

初回起動時にAnthropicアカウントとの認証が求められます。画面の指示に従って認証を完了してください。

プロジェクトフォルダの設計も重要です。複数クライアントのSEO業務を扱う場合、以下のようにクライアントごとにフォルダを分離し、それぞれにCLAUDE.mdを配置する構成が実運用では機能します。

```
seo-workspace/
├── CLAUDE.md              # 共通ルール（分析基準、出力フォーマット等）
├── client-a/
│   ├── CLAUDE.md          # クライアントA固有の情報
│   ├── analysis/          # 分析レポート
│   └── articles/          # 記事ファイル
├── client-b/
│   ├── CLAUDE.md
│   ├── analysis/
│   └── articles/
└── skills/                # 共通スキル定義
    ├── seo-query-research/
    ├── seo-structure/
    ├── seo-writing/
    └── seo-rewrite/
```

### CLAUDE.mdの設計ポイント

CLAUDE.mdは「Claude Codeへの業務引き継ぎ書」です。ここに書いた内容がClaude Codeの振る舞いを決定づけるため、設計の良し悪しがアウトプットの質に直結します。

SEO業務向けのCLAUDE.mdで最低限定義すべき項目は以下の4つです。

**役割定義**: Claude Codeにどんな立場で振る舞ってほしいかを明記します。「あなたはSEOコンサルタントのパートナーとして、データ分析・記事制作・レポート作成を担う」といった形です。

**クライアント情報**: 担当サイトのドメイン、ターゲット市場、競合企業、事業KPIなどを記載します。Claude Codeがこの情報を踏まえて分析や提案を行うため、具体的であるほど精度が上がります。

**分析ルール**: Ahrefsデータの取り扱い（金額はUSDセント単位で返却されるため100で割る、など）、優先度の判定基準（検索Vol 5,000以上かつ10位以内を「高」とする、など）を定義します。

**出力フォーマット**: レポートの構成、ファイル命名規則、保存先ディレクトリを統一しておくと、後から整理する手間が大幅に減ります。

```markdown
# CLAUDE.md 例（抜粋）

## 役割
あなたはSEOコンサルタントのパートナーです。
データ分析、記事制作、レポート作成を担当してください。

## クライアント情報
- ドメイン: example.co.jp
- 事業領域: BtoB SaaS（プロジェクト管理ツール）
- 主要競合: competitor-a.com, competitor-b.com
- KPI: オーガニックセッション月間50,000達成

## Ahrefs利用時の注意
- traffic_value等の金額データはUSDセント単位。表示時は100で割ってUSD換算する
- 初回利用時に必ずdocツールでAPIドキュメントを確認する
- where句はJSON形式で記述する

## 出力ルール
- 分析レポートは analysis/ フォルダに保存
- ファイル名: {分析種別}-{YYYY-MM-DD}.md
- 日本語で出力する
```

### MCP接続の設定方法

MCPの設定は、Claude Codeの設定ファイル（`~/.claude/settings.json` または `.claude/settings.json`）に記述します。

Ahrefs MCPの導入は以下の手順です。

```bash
# Ahrefs MCP サーバーを追加
claude mcp add ahrefs -- npx -y @ahrefs/mcp
```

Ahrefs MCP利用時の実務的な注意点として、初回は必ず`doc`ツールでAPIドキュメントを確認してからクエリを実行してください。利用可能なエンドポイントやパラメータの仕様がバージョンによって異なるためです。また、同時に大量のクエリを投げるとレートリミットに引っかかることがあるため、順次実行が安全です。

GSC連携はスクリプト経由で実現します。Google Cloud ConsoleでSearch Console APIを有効化し、サービスアカウントを作成した上で、取得したJSONキーファイルをプロジェクトに配置します。Claude Codeはこのスクリプトを呼び出す形でGSCデータを取得します。

Slack MCPを追加すれば、クライアントとのやり取りの文脈をClaude Codeが直接参照できるようになります。「直近のSlackでどんな議論があったか」を踏まえた提案が可能になるため、特にコンサルティング業務では効果的です。

## スキルでSEOワークフローを自動化する

MCP接続でデータへのアクセスが整ったら、次は繰り返し実行するワークフローを「スキル」として定義します。スキルはClaude Codeの中核機能の一つで、SEO業務の品質と効率を安定させる鍵になります。

### SEOスキルの設計思想

スキルとは、Markdownファイルに作業手順・判断基準・出力フォーマットを記述したワークフロー定義です。プログラミングは不要で、日本語でMarkdownが書ければ誰でも作成できます。

スキル化する意義は3つあります。**再現性**——同じ手順を毎回ブレなく実行できる。**品質安定**——判断基準を明文化しておくことで、分析の抜け漏れや品質のバラつきを防げる。**チーム展開**——作成したスキルを共有すれば、チームメンバーが同じ品質でSEO業務を回せる。

スキルファイルはプロジェクトの `.claude/skills/` ディレクトリに配置します。ファイル名を `SKILL.md` にしておくと、Claude Codeがスラッシュコマンドとして自動認識します。

### 記事制作4工程スキルの実装例

SEO記事制作では、以下の4つのスキルを順番に実行するフローを構築しています。

**工程1: /seo-query-research（クエリ調査）**

対象キーワードについて、検索意図の分類、ユーザーの悩み、上位5サイトの分析、関連ロングテールキーワードの抽出を行います。Web検索を活用して実際の検索結果上位のコンテンツを分析し、客観的なデータに基づく調査レポートを生成します。

出力は構造化されたフォーマットで、ターゲット読者、検索意図、解決すべき課題、関連キーワード20個などが整理されます。

**工程2: /seo-structure（構成案作成）**

クエリ調査の結果を受けて、上位記事との差別化戦略を設計し、記事の見出し構成を作成します。H2を5〜8個、必要に応じてH3を配置し、各セクションで扱うべき情報ポイント・事例・図表の提案まで含めた構成案を出力します。

競合分析では、上位5サイトがカバーしているトピック、情報の深さ、事例の新しさ、解決策の実用性を評価し、それらの不足を埋める差別化ポイントを特定します。

**工程3: /seo-writing（記事執筆）**

構成案に基づいて、10,000文字を目安にSEO記事を執筆します。スキル内に文章表現ガイドラインを詳細に定義しており、表現の多様性、感情表現の活用、段落構成のルールなどに従って執筆が進みます。

見出し直下には必ず100文字以内のリード文を配置する、箇条書きの1項目は160文字以内にする、といった細かなルールも定義されているため、一定の品質水準を維持したアウトプットが安定して得られます。

**工程4: /seo-rewrite（リライト）**

AI生成文章特有のクセ——テンプレート感、説明書感、記号の多用、過剰な丁寧さ——を取り除き、人が書いたと感じる自然な日本語に全面的に書き直します。

リライトスキルには「厳守ルール」が20項目以上定義されています。内容の捏造をしない、安全クッション表現を削除する、文のリズムを均一にしない、コロンやスラッシュを多用しない——こうしたルールに従って、BtoB記事として適切なトンマナに仕上げます。

この4工程を1つのセッション内で連続実行すると、各工程の出力が次の工程に自動で引き継がれます。キーワード調査の結果が構成案に反映され、構成案の見出しが本文執筆のガイドになり、執筆された文章がリライトの入力になる——手動でのデータ移行が一切不要です。

### 分析系スキルの実装例

記事制作だけでなく、分析業務もスキルとして定義できます。

**/seo-recovery-plan** は、サイト全体のSEOデータを分析し、トラフィック回復のための具体的なリライト計画を策定するスキルです。Ahrefs MCPからキーワードポートフォリオ、トップページ、トラフィック推移を取得し、記事ごとの優先度（高・中・低）を判定します。判定基準は検索ボリュームと現在順位の組み合わせで定義されており、CTR目安（1位: 25-30%、2位: 12-15%、3位: 8-10%）に基づくインパクト試算まで自動で行います。

**/seo-change-brief** は、公開済みの記事URLを入力すると、リライトの変更指示書を自動生成するスキルです。現状分析→問題点特定→具体的な修正指示という構成で出力され、ライターやチームメンバーに渡せる形式になっています。

## 実践ワークフロー — キーワード調査から効果測定まで

構築した基盤を使って、実際のSEO業務がどう流れるかを時系列で追います。

### キーワード調査フェーズ

新規記事のキーワードを選定する場面を例に取ります。Claude Codeのターミナルで以下のように指示します。

```
/seo-query-research SaaS管理ツール 比較
```

このコマンドを実行すると、まずWeb検索で「SaaS管理ツール 比較」の上位10件を分析し、検索意図（比較検討型）、ユーザーの悩み（ツールが多すぎて選べない、導入コストが不透明）、上位記事のカバー範囲を抽出します。

Ahrefs MCPが接続されている環境では、ここにキーワードの検索ボリューム（月間推定）、キーワード難易度、CPC、関連キーワードの定量データを重ねることができます。Web検索で得られる定性的な洞察と、Ahrefsの定量データを組み合わせることで、より精度の高いキーワード選定が可能になります。

### 記事制作フェーズ

キーワードが決まったら、同じセッション内で構成案の作成に進みます。

```
/seo-structure SaaS管理ツール 比較
```

上位記事の分析結果を踏まえ、差別化戦略を立てた上でH2/H3の見出し構成が出力されます。各セクションには「含めるべき情報ポイント」「使用すべき事例やデータ」「挿入すべき図表の提案」が付記されるため、次の執筆工程がスムーズに進みます。

続けて執筆とリライトを実行します。

```
/seo-writing SaaS管理ツール 比較
/seo-rewrite
```

4工程を通した所要時間は、テーマの複雑さにもよりますが、10,000文字の記事で概ね1〜2時間程度です。手作業で同じ品質の記事を制作する場合に2〜3日かかることを考えると、大幅な時間短縮になります。

### 効果測定・改善フェーズ

記事を公開した後は、GSCデータを使って効果測定を行います。

Claude Codeからスクリプトを実行し、公開後1ヶ月・3ヶ月の時点でクリック数、表示回数、平均掲載順位、CTRの推移を取得します。目標順位に到達していない記事や、表示回数に対してCTRが低い記事を特定し、改善施策を立案します。

サイト全体のトラフィックが下降傾向にある場合は、`/seo-recovery-plan` でリカバリープランを自動生成します。どの記事を、どの優先度で、どのようにリライトすべきかが、データに基づいて整理されたプランが出力されます。

月次レポートもClaude Code上で効率化できます。GSCとAhrefsのデータを統合し、主要KPIの推移、順位変動のあったキーワード、新規獲得キーワードなどをまとめたレポートを自動生成する運用が可能です。

## 導入時のつまずきポイントと対策

Claude CodeでSEO基盤を構築する過程では、いくつかのつまずきポイントがあります。事前に知っておくと回避しやすくなります。

### よくあるエラーと解決方法

**MCP接続エラー**: Ahrefs MCPでもっとも多いのは、APIキーの期限切れとレートリミットの超過です。APIキーは定期的に有効性を確認し、大量データの取得時は順次実行でリクエストを分散させてください。

**コンテキスト不足**: スキルを実行したのに期待どおりの出力が得られない場合、CLAUDE.mdの情報が不足しているケースが大半です。特にクライアント固有の専門用語やNG表現がある場合は、CLAUDE.mdに明記しておく必要があります。

**Ahrefsデータのフォーマット問題**: traffic_valueなどの金額データはUSDセント単位で返却されます。「トラフィック価値が235,000」と表示されたら、実際は2,350 USDです。この変換ルールをCLAUDE.mdに書いておけば、Claude Codeが自動的に換算して表示します。

### コスト管理の考え方

Claude Codeの利用にはAnthropic APIの料金がかかります。SEO業務で使う場合のコスト感を把握しておくと、運用計画が立てやすくなります。

Claude Codeでは複数のモデルを使い分けることができます。単純なファイル検索やフォーマット変換にはHaiku（最安）、記事生成・分析・リサーチにはSonnet（中程度）、高度な戦略立案や複雑な判断にはOpus（最高）を割り当てる——というルールを設けることで、品質を落とさずにコストを最適化できます。

実務的な目安として、10,000文字のSEO記事を4工程（調査→構成→執筆→リライト）で制作する場合、Sonnetベースで数百円〜1,000円程度のAPI料金です。手作業にかかる人件費と比較すれば、コストパフォーマンスは十分に高いと言えます。

## まとめ

Claude CodeによるSEO分析基盤は、一度構築してしまえば、日々のSEO業務の効率と品質を大きく変えます。

最初からすべてを揃える必要はありません。CLAUDE.mdとWebSearchだけの最小構成から始め、業務の中でAhrefs MCPやGSC連携、スキルを一つずつ追加していく——この段階的なアプローチが、結果的にもっとも定着しやすいやり方です。

ツールの使い方を覚えるのではなく、自分の業務フローに合わせてClaude Codeを設計する。この発想の転換が、AI活用で成果を出すための分かれ目になります。
