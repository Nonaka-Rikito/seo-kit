# 【2026年最新】GA4のタグ設定で見落としやすい5つの設定ミスと対策方法

「GA4を導入したけれど、思うようにデータが取れていない」
「レポートの数値がおかしい気がするが、原因がわからない」

GA4（Google Analytics 4）への移行が完了しても、タグ設定に不備があると、正確なデータ収集ができず、分析結果の信頼性が大きく損なわれます。実際に、私たちがクライアントのGA4環境を監査すると、ほぼすべてのケースで何かしらの設定漏れが見つかります。

この記事では、GA4のタグ設定で企業が陥りやすい5つの設定ミスを取り上げ、それぞれの原因と具体的な修正手順を解説します。自社のGA4設定を見直すチェックリストとしてご活用ください。


## GA4のタグ設定を正確に行うべき理由

GA4はユニバーサルアナリティクス（UA）から大きくアーキテクチャが変わり、イベントベースの計測モデルを採用しています。UAの時代に正しく設定できていたとしても、GA4では別の設定が必要になるケースが少なくありません。

タグ設定の不備が引き起こす具体的な問題は以下の通りです。

・コンバージョン数の過少計測 → 広告運用の判断を誤る
・ユーザー属性データの欠損 → ターゲット分析の精度が落ちる
・重複データの計測 → レポートの数値が実態と乖離する
・セッションデータの断絶 → ユーザー行動の正確な把握ができない

これらの問題は、売上やマーケティングROIの判断に直結します。「データは取れているから大丈夫」と思い込まず、設定内容を定期的に見直すことが、データドリブンな意思決定の土台になります。


## 設定ミス1：イベントトラッキングの設定不足

GA4で最も多い設定不備が、イベントトラッキングの設定漏れです。


### なぜこの問題が発生するのか

GA4では、ページビューだけでなく、ボタンクリック、フォーム送信、ファイルダウンロード、動画再生など、ユーザーのあらゆる行動を「イベント」として計測します。

しかし、多くの企業では以下の理由からイベントトラッキングが不十分なまま運用されています。

・「どのユーザー行動を計測すべきか」の事前設計ができていない
・GA4のイベント設定方法（GTM経由 vs GA4管理画面）の違いを理解していない
・自動収集イベントだけで十分だと思い込んでいる

GA4には「自動収集イベント」「拡張計測機能イベント」「推奨イベント」「カスタムイベント」の4種類があります。自動で取れるデータには限りがあり、ビジネスに必要な行動データの多くは手動設定が必要です。


### 具体的な対策手順

1. 自社のコンバージョンポイントを洗い出す
   例：資料請求、問い合わせフォーム送信、会員登録、購入完了、電話ボタンタップなど

2. 各コンバージョンポイントに対応するイベントが計測されているか確認する
   GA4の管理画面 → レポート → エンゲージメント → イベントで、該当イベントが存在するか確認

3. 不足しているイベントをGoogle Tag Manager（GTM）で設定する
   GTMのトリガーとタグを使って、クリックイベントやフォーム送信イベントを作成

4. GA4のリアルタイムレポートで正しく計測されているかテストする
   設定後は必ず実機でテストし、イベントが正しく発火するか確認

5. ビジネス上重要なイベントは「コンバージョン」としてマークする
   GA4管理画面 → コンバージョン → 新しいコンバージョンイベントで、キーイベントを登録


## 設定ミス2：ユーザープロパティの不適切な設定

ユーザープロパティは、サイト訪問者の属性情報を追跡するための機能です。これが適切に設定されていないと、セグメント分析やオーディエンス作成の精度が低下します。


### よくある問題パターン

・会員ステータス（無料/有料）や利用プラン情報が設定されていない
・ユーザープロパティの命名規則がバラバラで、レポートが混乱している
・不要なプロパティが大量に設定され、上限（25個）を圧迫している
・個人を特定できる情報（氏名、メールアドレスなど）を誤って設定している（Googleポリシー違反）


### 具体的な対策手順

1. 分析に必要なユーザー属性をリストアップする
   例：会員種別、利用プラン、登録日、利用端末、地域

2. GA4のユーザープロパティ上限（25個）を踏まえ、優先順位をつける

3. 命名規則を統一する
   推奨：スネークケース（user_type、subscription_plan など）

4. GTM経由でユーザープロパティを設定する
   GA4設定タグの「ユーザープロパティ」セクションにキーと値を追加

5. 個人情報に該当するデータが含まれていないか確認する
   氏名、メールアドレス、電話番号などの送信はGoogleのポリシーで禁止されているため、絶対に避ける

6. 四半期ごとに使用中のプロパティを棚卸しし、不要なものは削除する


## 設定ミス3：UA（ユニバーサルアナリティクス）からGA4への移行時の設定引き継ぎ漏れ

2023年7月1日にUAのデータ処理が停止して以降、すべてのサイトがGA4での運用に切り替わっています。しかし、移行から時間が経った今でも、UA時代の設定が正しくGA4に引き継がれていないケースが散見されます。


### 移行時に見落とされやすいポイント

・UAのゴール設定がGA4のコンバージョン設定に変換されていない
・UAで使っていたカスタムディメンションがGA4で再設定されていない
・クロスドメイン計測の設定が移行されていない
・フィルタ設定（社内IPの除外など）がGA4側で未設定
・eコマース計測のデータレイヤー構造がGA4仕様に更新されていない


### 具体的な対策手順

1. UAの設定一覧をエクスポートする
   UA管理画面の「ビュー」設定から、ゴール、フィルタ、カスタムディメンションをスクリーンショットまたはスプレッドシートに記録

2. GA4の現在の設定と突き合わせる
   GA4管理画面で、コンバージョン、カスタムディメンション、データストリーム設定を確認

3. 不足している設定を一つずつ追加する

4. 特にクロスドメイン計測は見落とされがちなので、別ドメインを運用している場合は優先的に確認する
   GA4管理画面 → データストリーム → タグ設定を行う → ドメインの設定

5. 社内IPの除外フィルタを設定する
   GA4管理画面 → データストリーム → タグ設定を行う → 内部トラフィックの定義


## 設定ミス4：複数GA4プロパティの重複設定

意外と多いのが、同じWebサイトに複数のGA4プロパティが設定されてしまっているケースです。


### なぜ重複が発生するのか

・移行時に新旧のプロパティが両方残っている
・異なる部署やベンダーがそれぞれプロパティを作成した
・テスト用プロパティが本番環境に残ったまま
・GTMとページ直書きのタグが併存している

重複プロパティがあると、同じユーザーの行動が二重にカウントされ、セッション数やイベント数が実際の2倍近い数値になることがあります。これはレポートの信頼性を根本から損ないます。


### 具体的な対策手順

1. ブラウザの開発者ツール（F12）を開き、Networkタブで「collect」を検索する
   GA4に送信されるリクエストに含まれる測定ID（G-XXXXXXXXXX）を確認

2. 複数の測定IDが見つかった場合、どちらが正規のプロパティか特定する

3. 不要なプロパティのタグを削除する
   GTMを使っている場合：該当するGA4設定タグを無効化または削除
   ページ直書きの場合：HTMLソースから該当するgtag.jsコードを削除

4. Tag Assistant（Chrome拡張機能）を使って、削除後に正規タグのみが発火していることを確認

5. 今後の運用ルールとして「GA4プロパティの作成・管理は特定の担当者に集約する」ことを社内で決める


## 設定ミス5：データストリームの設定不足

GA4のデータストリームは、WebサイトやアプリからGA4へデータを送信するためのパイプラインです。この設定が不十分だと、特定のサイトやアプリのデータが収集されません。


### よくある問題パターン

・新しくサブドメインやランディングページを追加した際に、データストリームが未設定
・拡張計測機能（Enhanced Measurement）がオフになっている
・アプリ（iOS/Android）のデータストリームが未設定
・データストリームの測定IDがGTMやHTMLに正しく反映されていない


### 具体的な対策手順

1. GA4管理画面 → プロパティ → データストリームで、現在のストリーム一覧を確認する

2. 自社が運用しているすべてのWebサイト・アプリが網羅されているか照合する

3. 不足しているサイト・アプリのデータストリームを新規作成する

4. 各データストリームの「拡張計測機能」がオンになっていることを確認する
   特に「ページビュー」「スクロール」「離脱クリック」「サイト内検索」「フォーム操作」は有効にしておく

5. 新しいサイトやページを公開する際のチェックリストに「GA4データストリームの設定確認」を追加し、漏れを防ぐ


## GA4タグ設定の定期監査チェックリスト

ここまで紹介した5つの設定ミスを踏まえ、月次・四半期で実施すべき監査項目を整理します。

【月次チェック】
・リアルタイムレポートでデータが正常に流入しているか
・主要コンバージョンイベントが正常に計測されているか
・異常値（急激なトラフィック増減、0件のイベントなど）がないか

【四半期チェック】
・イベント一覧の見直し（不要なイベントの整理、新規イベントの追加検討）
・ユーザープロパティの棚卸し
・データストリームの網羅性確認
・クロスドメイン計測やIPフィルタが正常に機能しているか
・GTMのコンテナバージョン管理（不要なタグの整理）

このチェックリストをスプレッドシートに落とし込み、チーム内で共有・運用すれば、タグ設定の品質を継続的に維持できます。


## まとめ：GA4のタグ設定を今すぐ見直すための3ステップ

GA4のタグ設定は「一度設定したら完了」ではなく、ビジネスの変化やGA4のアップデートに合わせて継続的にメンテナンスすべきものです。

今すぐ着手すべき3つのアクションを挙げます。

1. ブラウザの開発者ツールで自社サイトのGA4タグを確認し、重複プロパティがないかチェックする（所要時間：5分）

2. GA4管理画面のイベント一覧を開き、自社のコンバージョンポイントがすべて計測されているか照合する（所要時間：30分）

3. 上記の月次・四半期チェックリストを作成し、次回のチーム定例で運用ルールを決める（所要時間：1時間）

正確なデータ収集は、すべてのマーケティング施策の出発点です。この記事を参考に、自社のGA4環境を改めて見直してみてください。
