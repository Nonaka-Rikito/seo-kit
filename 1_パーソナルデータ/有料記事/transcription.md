それらすべてをClaude Codeに集約し、毎朝AIに工程表を作ってもらい、プロジェクトの背景情報を加味した戦略立案、トレンド収集から記事執筆、タスク管理までをコマンド一発で回す。

**もはやClaude Codeネイティブで仕事をしているかどうかで、非エンジニアの生産性にかなりの差が出る時代になった**と感じています。

きっかけは、先日久々に会ったAI Nativeのたなまこさんでした。

彼はClaude Codeのヘビーユーザーで、仕事で使うツールをすべてClaude Codeと連携し、Claude Codeとだけ対話するというスタイルで仕事をしています。

これに刺激を受けて一気にClaude Codeへの集約を進めたところ、数日で「AI経営管理＆業務遂行システム」と呼べるものが出来上がりました。

本記事では、その全体像と実際にどう使っているのかの具体を共有します。

さらに、**この記事の内容をもとにした「構築マニュアル」のMarkdownファイルも配布しています。**

**これをClaude Codeに渡して「この通り構築お願い」と頼むだけで、このnoteで紹介したような仕事環境を良い感じに構築してくれます。**

記事を読んで「やってみたい」と思ったら、自分でゼロから設定する必要はなく、Claude Codeにマニュアルを渡して、対話するだけで同じような仕事環境をつくることができます。

**忙しくて記事読む暇がないよ！という方はこのmdをClaude Codeに渡すだけでも大丈夫です！笑**

## 全体アーキテクチャ：事業全体を扱う1つのフォルダをつくり、それをAIの「脳」にする

具体的にまず何をしたかというと、経営関連のファイルやプロジェクトドキュメントなど、諸々のドキュメントをひとつのフォルダに集約し、そのフォルダをClaude Codeで開けるようにしました。

Claude Codeはプロジェクトフォルダ全体をAgentic RAG（AIが必要に応じてフォルダ内のファイルを自律的に検索・参照する仕組み）で読みにいくことができます。

つまり、ここに必要な情報をすべて配置しておけば、AIが文脈に応じて適切なファイルを自分で見つけて読み込んでくれるため、擬似的な「長期記憶」として機能します。

実際のディレクトリ構成は以下の通りです。

[画像: ディレクトリ構成のスクリーンショット]

**このフォルダ全体がAIにとっての「業務コンテキスト」として機能しています。**

ちなみに、Notionなどからのドキュメント移行も、Claude CodeにNotion MCPを連携させて、Claude Codeに移行して欲しいNotionディレクトリを伝えたら一発でした。

また、今まで用途ごとにChatGPTやGeminiなど多種多様なAIツールを使い分けたりしていましたが、それらのAPIキーをClaude Codeに渡して後述のサブエージェントを作成することで、ツールを切り替えずにClaude Codeだけで完結する環境を構築しました。

これによってツールを切り替える必要がなくなって認知的な負荷がだいぶ下がった感覚を持っています。

## CLAUDE.md：「最重要マニュアル」に書いていること

全体フォルダの直下にある「CLAUDE.md」というファイルに、Claude Codeに常に理解しておいてほしい「最上位の業務マニュアル」を書き込んでおり、Claude Codeはセッション開始時にこのファイルを読み込んで「経営パートナー」として振る舞えるようになります。

CLAUDE.mdには、以下のような指示が書かれています。

- **役割定義**: 「私の経営パートナーとして、事業戦略の壁打ち、サービスの設計、リサーチなどをサポートしてください」
- **ディレクトリごとの役割説明**: 各フォルダが何を意味し、どういう情報が入っているか
- **よく使う外部ツール連携の設定**: Notion、Google Calendar、Google Sheetsとの接続方法
- **ワークフロー定義**:
  - 「おはよう」「今日の予定は?」と打つと → /daily-scheduleスキルが自動発動
  - 「Issueにして」「タスク登録して」と言うと → /issue-triageスキルが自動発動
  - 「覚えておいて」と言うと → /agent-memoryが発動
- **コンテンツパイプライン定義**: 情報収集→分析→発信→効果測定→次の企画の流れ
- **連携しているGitHub Issueワークフロー**: カンバンボード、WIP上限、ラベル体系の定義

要するに、**CLAUDE.mdはClaude Codeという一社員の行動原理であり、自社に適切にオンボードするための「最重要業務マニュアル」**です。

一度書いておけば、Claude Codeは毎回このマニュアルを読み込んだ状態で仕事を始めてくれます。

## Skillsで複雑な業務フローに再現性をもたせる

Claude Codeには「Skills（スキル）」という機能があります。

「どんな手順で、何を参照し、何を出力するか」という業務プロセスをMarkdownで定義しておくと、以降そのプロセスが必要なタイミングをClaude Code自体が自律的に判断してその手順通りに実行してくれます。

つまり、**複雑な業務フローに再現性をもたせる業務別マニュアルがSkills**です。

ちなみに、**Skillsの作成自体もClaude Code任せでOK**です。

1. **自分**: 「こういう作業をやって」とClaude Codeに依頼する
2. **Claude Code**: 必要な設定を行いながら作業を実行してくれる
3. **自分**: 出来上がった作業フローを見て「これをSkills化して」と依頼する
4. **Claude Code**: Skillsファイルとして保存し、以降は関連する依頼があった際に自動で起動するようになる

[画像: Skillsの概念図 - 01 スキル定義 → 02 自動判断・実行 → 03 ファイル出力 → 04 フィードバック]

つまり、**一度やってもらった作業を「再現可能な業務プロセス」に変換する**だけです。

プログラミングの知識は一切不要です。

現在、複数のスキルを運用していますが、非エンジニアリング業務でも役立つものが多いです。いくつか紹介します。

### 朝の工程表：/daily-schedule

毎朝「おはよう」と打つだけで、このスキルが自動で起動します。実行される処理は以下の通りです。

「今日何をすべきか」と時間毎のタスクスケジュールをClaude Codeに教えてもらうようにしています。

[画像: 2026-02-06（木）の工程表]

秘書さんがその日の予定とやるべきことを教えてくれる感覚で、だいぶ仕事をするのが楽になりました。

▼ Skillsのフローは以下のような流れ

1. Google Calendarから今日の予定を取得（MCP連携で、メインカレンダーとルーティンカレンダーの両方）
2. GitHub Projectsから進行中のタスクを取得
3. 昨日の日報（DAILY.md）を読み込み、未完了タスクを確認
4. プロジェクトファイルと経理データを確認
5. タスクを「緊急度×重要度」と「認知負荷」で分類
6. 15分刻みの工程表を生成し、ファイルに自動保存

しかも、この「秘書」は全プロジェクトの進捗、財務状況、カレンダー、昨日の作業内容をすべて把握した上で優先順位を判断してくれます。

人間の秘書でもここまでのコンテキストを毎朝把握するのは難しいでしょう。

### コンテンツパイプライン：情報収集→分析→発信→効果測定

コンテンツ発信に関連するスキルだけで7個あり、これらが連鎖してパイプライン全体を回します。

1. **/note-trend-daily** — X（Twitter）投稿、note.com、Issue経由のいずれかでトレンド情報を収集し、「週刊」や社名テーマ（例：「渋谷で4席」等）で「ウォッチリスト」に自動蓄積

2. **/digest** — たとえば4つの本の抄訳を作り、レポートと比較文を生成

3. **/note-intro-analyze** — 記事ごとの初動パフォーマンスの分析。エンゲージメント数のコンテンツ別の推移やYouTubeスクリプトに関連する記事を出力

4. **/generate-seo** — Gemini AIやahrefs APIを用い、note記事の製作（企画立案含む記事のすべて）

5. **/title-gen** — 記事タイトルを複数パターン生成

6. **/cover-image-prompt** — 記事カバー画像のMidjourneyプロンプトを生成

ほか：情報蓄積→分析→発信→効果測定→次の企画というコンテンツパイプラインのスキルが、スキル連結が可能で柔軟です。

ちなみに、この記事自体も /generate-seo スキルを使って下書きを出させ、手を入れながら作成しています。

### タスク・記憶管理

- **/issue-triage** — 「事業のワークショップの準備をするよ」といえばgit、GitHub Issueを自動で発行し、適切なラベル（/priorities, priority:high, review-notesなど）をつけてカンバンに配置

- **/agent-memory** — 「覚えておいて」と言えば、業務方針、個人的な好み、過去の決定事項をKnowledge Baseに永続的に保存。次のセッションから参照される

- **/update-finance** — 経理スプレッドシートの自動更新データを同期してくれる

一度機能してしまえば、日々の動詞は「おはよう」「トレンドチェック」「これをIssueにして」「覚えておいて」だけ。プロセスの詳細をいちいち指示する必要がありません。

## Agent Team：5体のAIによる戦略分析

最近リリースされたClaude CodeのAgent Team（マルチエージェント）も活用しています。

例えば、note記事の執筆時に、「AI編集室」のようなチームを構築しています。

常設化ではなく必要に応じて都度スキルを呼ぶ設計になっています。

▼ 編集チームのワークフロー

[画像: 編集チームのワークフロー図]
- 戦略リサーチ・エージェント
- 構成エージェント
- 執筆エージェント
- 品質レビュー・エージェント
- 編集長（メイン）

記事のテーマを投げると、まず5体のリサーチ・アナリスト・エージェントが並列で分析を走らせます。

角度から深掘りをしていきます。

1チームは競合環境の分析、1チームは事業ベースの改善調査、1チームはまとめて5つの記事を提示してくれる仕組みです。

はたしてリサーチ段階で5体の検索Agentコンシェルジュが並列して記事構成の情報を収集し（「5つ薦めてくる」タイプのリサーチ分担）指示書を出力しつつ、その後のセクションライティングについても分担して進行、さらに品質レビューまでのAIチェーンとして機能しています。

そしてファイル・ベーシック・ストラテジストがGemini APIを使いつつ戦略を作成し、完成する仕組みになっています。

このように本来は複雑な議論をしなければならなかったベーシック・ストラテジー層にもこのマルチレーションシステム(Claude CodeのAgent Team構造）が導入されてつくる手間がかなり楽になりました。

## MCP連携：外部ツールとの「記憶」

Claude Code単体でもかなりのことができますが、MCP（Model Context Protocol）連携によって外部ツールとシームレスにつながることで、さらに能力が拡張します。

連携しているツールの一部を紹介します。

[画像: MCP連携図 - Claude Code を中心としたツール連携の図]

- **Google Calendar** — 予定の取得・スケジュール管理（メインカレンダー・ルーティンカレンダー）
- **Notion** — 事業のドキュメント・データベースの読み書き（カンバンやマスタ・タスク管理）
- **Google Sheets** — 経理スプレッドシートの読み取り
- **Slack** — メッセージの送受信（Slackへの即時調整をするSlackボットプリフライト含む）
- **GitHub** — Issue管理（Life ManagementプロジェクトやIssueの振り分け）
- **Figma** — デザインファイルの制御型デザインレビュー（クラウドAIベースのアクションを配置）
- **Playwright** — Web自動化・スクリーンショット・OCR

加えて、vAI（GoAI）のAPIをPythonスクリプト経由で呼び出して検証検索の精度チェックもGemini APIアクセスプロンプトで実施しています。

さらに、既存のMCPに欲しい機能がない場合も、**Claude Codeに「このツールと連携するMCPを作って」と指示するだけで、カスタムMCPサーバーも構築してくれます。**

実際に、自分が普段使っているアプリやWebサブスクリプション情報をClaude Codeから参照できるようにするためのカスタムMCPも作りました。

**重要なのは、これらの連携のすべてをClaude Codeと対話するというところで完結しているということです。**

以前はChatGPTで概要をつくり、Geminiで具体を調べ、Notionで管理し、Google Sheetsを手動で更新、という流れでしたが、この全プロセスが1つのフォルダ・1つのインターフェースに集約されました。その認知的な切り替えコストがゼロになる価値は、特別なAIの性能差以上に大きいと思っています。

## 構築自体もClaude Codeとの対話で完結する

ここまで読んで、「これはエンジニアの技術知識がないと構築できないのではないか」と思うかもしれませんが、構築に関しては初期構築はディレクトリ設計やMakemeetでのスキル定義など、一定のリテラシーは必要です。

しかし、ここで重要な事実があります。**このシステムの初期構築自体も、Claude Codeとの対話ですべて完結しました。**

「毎朝の工程表を自動生成するスキルがほしい」と言えば、Claude CodeはSkillsファイルの作りかた含めて丸ごと作成。「Google Calendarと連携させて」と言えば、MCP設定の定義を含めて、「経理データをGoogleシートから引っぱりたい」と言えば、必要なコードもスキルもまとめて生成してくれます。

つまり、Claude Codeが「覚えるものをセットアップ」。前者をClaude Codeに手渡しできるツール、です。

一度構築してしまえば、あとは日々のやりとりだけで運用を回すだけです。

## なぜ「エンジニア以外」にこそClaude Codeなのか

ここからが平実等確信の部分です。

エンジニアにとってのClaude Codeは、主にコーディング支援ツールです。

コードを書き、テストし、プロンプトを打ち、さまざまにテストが通るとその先の大きな価値を生み出すまで、複雑を反復的な実行をするためのワークフローの「効率化」という位置づけです。

一方、非エンジニアにとってのClaude Codeは**「今まで存在しなかった "自社の事業を把握する全能秘書AIツール"」**です。

- 社内の複数情報源から横断的にコンテキストを把握して判断してくれるAI
- 初社の経営情報やリサーチ結果・分析してくれるAIリサーチャー
- 自分の仕事のフローを配置的に実行してくれるコンサルティングチーム
- コンテンツの制限調査から企画を提案してくれるAIマーケター
- 事業に関する記憶と引き出しを管理してくれるAIメモリーシステム

これらの優先案、入手可能で、複数のToolを統合的にClaude Codeで活用してくれるという強力なツールだということです。トータルのClaude Code FIle、Markdownファイルとして記述するだけです。

## 実運用で感じた3つのこと

### 1. 認知負荷の劇的な軽減

以前は、ChatGPT、Gemini、Claude、Notion、Google Sheets、GitHub……と目標別にツールを使い分けていましたが、それぞれのツールにコンテキストスイッチが発生していました。あれしなきゃこれしなきゃ、ツール間の行ったり来たり、「あの記事のデータ」を探す手間もなくなり、特別に目の前のことに集中できるようになりました。

Claude Codeという1つのインターフェースに入力するだけで、必要なツールへのアクセスを裏側でやってくれることで、「入口のシンプルさ」は、生産性以上に、精神的な負荷と向き合っています。

### 2.「AIにマネジメントしてもらいながら働く」という感覚

新規事業にして抽象案をこなしていても、やるべきこともすべて完璧に把握し続けることは難しい。プロジェクトの情報・戦略・ベストプラクティスについて相談もでき、このことによる生産的な時間が増加する仕組みになっています。

もはや、AIは私より私のことを高い精度で理解して引き出してくれています。

### 3. GitHubによるマルチプラットフォームの恩恵

すべてのドキュメントをGitのポジトリで管理することで、全体的な変更履歴がバージョン管理され、変更差分の可視化フローが走り、リリースリクエスト（Pull Request）を配置することで、レビューワーカーを配置するなどの機能を実現しています。

## まとめ：Claude Codeは「プログラミングツール」ではなく「エージェントディレクションツール」

この記事で紹介したことの本質は、VPTやAIコードを書いた先の計算処理とは全く関係ありません。やっていることに2つにまとまります。

1. 手持ちのことを全てClaude Codeのディレクトリに集約する情報セットアップし、
2. Claude Codeに渡して、NotionやNotion側からすでにできていることをフル稼動して使う、
3. Claude Codeで集約できてパブリッシュできているフロントをClaude Code側にしてマルチエージェントを外して別サービスの仕組みを使う
4. GPTやGeminiで使っているプロンプトをClaude Code側にしてマルチエージェント方式で外部ツール連携をする
5. 外部ツールへの接続をClaude Codeにやってもらう

これはプログラミングではなくまさに**ディレクション**です。

自分のやりたいことと情報を自分でまとめ、構造を把握でファイルディレクトリにパッケージして、その使い方をClaude Codeに伝える。

**求められるのはコーディング能力ではなく、「自分の仕事の構造の理解を言語化する力」です。**

そしてこの能力は、経営者を含く全てのビジネスパーソンとエンジニアにおいても同じだと思います。

まだClaude Codeを使ったことのないけれどAIに関わるエンジニアのみなさん、コーディング以外の仕事にも使えることをぜひ、まずは手を付けてみてほしいと思っています。
