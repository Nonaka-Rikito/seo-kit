```# Claude Code モデル使い分けガイド

## 基本方針

**デフォルトは Sonnet。** タスクの重さに応じて使い分ける。

| モデル | 使いどころ |
|--------|-----------|
| **Haiku** | 要約・抽出・確認など明らかに軽いタスク |
| **Sonnet** | デフォルト（ほぼすべてのタスク） |
| **Opus** | 特に複雑・高品質が必要なとき（自動判断） |

---

## タスク別の目安

| タスク種別 | 推奨モデル |
|-----------|-----------|
| 記事・文章の生成・下書き | Sonnet |
| コードの設計・実装 | Sonnet |
| データ分析・レポート | Sonnet |
| 調査・リサーチ | Sonnet |
| 箇条書きの要約・単純な抽出 | Haiku |
| Yes/No の確認作業 | Haiku |
| 特に難易度の高い推論・複雑なコード設計 | Opus |

---

## 切り替え方法

### 起動時に指定する

```bash
claude --model claude-sonnet-4-6              # Sonnet（推奨デフォルト）
claude --model claude-haiku-4-5-20251001      # Haiku（軽いタスク）
claude --model claude-opus-4-6                # Opus（複雑なタスク）
```

### 設定ファイルでデフォルトを固定する

`~/.claude/settings.json` に記述：

```json
{
  "model": "claude-sonnet-4-6"
}
```

Task tool でサブエージェントを起動するときは `model` パラメータを明示する。

```
model: "haiku"   → 軽いタスク（調査・要約など）
model: "sonnet"  → 標準タスク（デフォルト）
model: "opus"    → 複雑なタスク（設計・高品質生成）

## 高橋さん文章生成ルール（報告文章・メッセージ生成時の必須参照）

報告文章・メッセージ・添削を生成する際は、以下の4ファイルを必ず事前に読み込んで準拠すること。

### 参照ファイル

`1_パーソナルデータ/高橋さん文章生成/` 配下：

| ファイル | 内容 |
|---------|------|
| `無題のドキュメント (1).md` | 言語ルールマスターDB完全版（ルール体系・添削手順・主要人物詳細・例文集） |
| `無題のドキュメント (2).md` | ルールDB生データ（全54件のTSV形式、全列データ） |
| `無題のドキュメント (3).md` | malna組織マスターデータ（会社概要・人物情報・クライアント略語・社内用語） |
| `無題のドキュメント (4).md` | ルール転置テーブル + 取引先マスター + 社員マスター |

### ルール適用ロジック

1. **対象者**（D列）×**用途**（E列）×**媒体**（F列）で該当ルールを検索
2. 優先度順にフォールバック: **100（人物ルール）→ 50（パターン）→ 30（媒体）→ 10（共通）**
3. **敬語レベル**（G列）を厳守（5=最敬体、3=標準丁寧体、2=フランク丁寧体）
4. **例文**（O〜Q列）のトーン・語尾・句読点を忠実に再現
5. **NG表現**（L列）は絶対に使用しない
6. **OK表現**（M列）を積極的に使用する

### 添削時の出力フォーマット

添削依頼の場合は、該当ルール→問題点→修正案→教育的フィードバックの構造で出力する（詳細はドキュメント(1)の添削テンプレートに準拠）。

- 長文レポートはファイルに書き出し、チャットでは要点サマリーを返す
