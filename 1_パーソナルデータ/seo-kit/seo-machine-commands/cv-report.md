# /cv-report - CV指標レポート

jo-katsu.com のCV指標（会員登録・資料DL）の日次・累計レポートを生成する。
CTAバナー変更がある場合はBefore/After効果測定も含める。

## パラメータ

`$ARGUMENTS` に対象月を指定（省略時は当月）。例: `2026-02`

## データソース

1. **GA4 Python API**: イベントデータ（`scripts/ga4_events.py`）
   - `pre_register_cta_click`: 会員登録イベント
   - `dl_material_page_view`: 資料DL申し込みページイベント
   - `/materials/` パスのPV: 資料申し込みページ遷移数
2. **config/cv-targets.json**: 月次目標値
3. **data/change_events/events.jsonl**: CTAバナー変更ログ
4. **mcp-gsc**: CTAバナー変更のBefore/After検索パフォーマンス比較

## 実行手順

### Step 1: 対象月の特定

- `$ARGUMENTS` があればその月を使用（例: "2026-02" → 2026年2月）
- 省略時は本日の年月を使用
- 対象月の開始日 = YYYY-MM-01
- 対象月の終了日 = 本日（月途中の場合）or 月末日

### Step 2: 目標値の確認

**目標値はスプシで管理されている。** 以下の順で目標値を決定する:

1. （任意）`spreadsheet_url` が設定されていれば、先に `scripts/sync_cv_targets_from_sheet.py` を実行してスプシから `config/cv-targets.json` を最新化できる。
2. `config/cv-targets.json` を読み込み、対象月キー（例: "2026-02"）を探す。
3. 対象月の設定がある場合はその値を使用。
4. **対象月の設定がない場合**:
   - `spreadsheet_url` が設定されていれば「スプシで最新の目標値を確認してください」と伝え、`sync_cv_targets_from_sheet.py` の実行または手動入力を促す。
   - 入力された目標値で `cv-targets.json` を更新してから続行。
5. `defaults` はあくまでフォールバック。新月度の初回実行時にユーザー確認を挟む。

### Step 3: GA4イベントデータ取得（4コマンド並行実行可）

**3a.** 会員登録イベント:
```bash
.venv/Scripts/python scripts/ga4_events.py event-daily pre_register_cta_click YYYY-MM-01 YYYY-MM-DD
```

**3b.** 資料DLイベント:
```bash
.venv/Scripts/python scripts/ga4_events.py event-daily dl_material_page_view YYYY-MM-01 YYYY-MM-DD
```

**3c.** 資料ページPV:
```bash
.venv/Scripts/python scripts/ga4_events.py pageviews-daily /materials/ YYYY-MM-01 YYYY-MM-DD
```

**3d.** オーガニックセッション数（CVR算出用）:
```bash
.venv/Scripts/python scripts/ga4_events.py organic-sessions [YYYY-MM-01] [YYYY-MM-DD] 10
```
→ サイト全体のオーガニックセッション数を取得。CVR（= CV数 ÷ セッション数）算出に使用する。

### Step 4: 累計計算とCVR算出

各イベントの日別データから:
- 各日の単日実績を集計
- 累計 = 初日からの合計
- 達成率 = (累計実績 / 月次目標) × 100（小数点1桁）
- 最終日（本日 or 月末）の単日実績を「本日の単日」として表示

**CVR（コンバージョン率）算出**:
- 会員登録CVR = 累計会員登録数 ÷ オーガニックセッション数 × 100
- 資料DL CVR = 累計DL数 ÷ オーガニックセッション数 × 100
- 前月同期間のCVRも算出し、CVR自体の増減を確認（流入減 vs CVR低下の切り分けに使用）

### Step 5: CTAバナー変更の確認

1. `data/change_events/events.jsonl` を読み込み（1行1JSON）
2. `type` が `"cta_banner"` のイベントをフィルタ
3. 対象月内の変更イベントがあれば:
   a. ベースラインデータ `data/change_events/evt-XXX_baseline.json` を読み込み
   b. 変更日から14日以上経過していれば効果測定を実施:
      - mcp-gsc `compare_search_periods` でBefore（変更前14日）vs After（変更後14日）
      - GA4イベントのBefore/After比較（可能な場合）
   c. 14日未満の場合は「測定中」と表示
4. 変更がない場合は「対象月のCTAバナー変更: なし」と記載

### Step 6: 要因分析（数値根拠を必ず示すこと）

CTAバナー変更のBefore/After結果について、以下の観点すべてを数値付きで分析する。
**「改善した/悪化した」だけの記述は禁止。必ず数値とその変動幅を示すこと。**

**6a. クエリ別CTR・順位変動分析**:
- mcp-gsc `get_advanced_search_analytics` で、変更対象ページの主要クエリ（上位10件）について Before/After のCTR・順位・クリック数を取得
- クエリごとに「CTR X.X% → Y.Y%（+Z.Zpt）」「順位 X.X → Y.Y」の形式で記載
- CTR変動が大きいクエリを特定し、その変動が「順位変動起因」か「訴求変更起因」かを判別する:
  - 順位がほぼ同じなのにCTRが変化 → 訴求（タイトル/ディスクリプション）変更の効果
  - 順位が大きく動いてCTRも連動 → 順位変動起因（訴求変更の直接効果ではない）
- **検索意図の分類**: 各クエリを以下のいずれかに分類し、CTA改善との関連度を評価する:
  - `transactional`（登録・申込み意図）: 「就活サイト 登録」「ES添削 無料」→ CTA改善の直接効果が出やすい
  - `commercial`（比較・検討意図）: 「就活サイト おすすめ」「就活エージェント 比較」→ CTA訴求とコンテンツ質の両方が影響
  - `informational`（情報収集意図）: 「ES 書き方」「面接 マナー」→ CTA改善より記事内CTA配置が重要
  - 検索意図がtransactional/commercialのクエリでCTR改善が見られない場合は、CTA以外の要因（コンテンツ品質、ページ速度等）を疑う

**6b. 訴求文言の変化比較**:
- `data/change_events/events.jsonl` に記録された変更内容（before/after）を引用
- CTAバナーのコピー文言・デザイン要素の変更前/変更後を並べて比較
- 「何をどう変えたから、どの数値がどう動いたか」の因果関係を明記

**6c. 外部要因の分離**:
- Ahrefs MCP `site-explorer-organic-keywords` で同ページの主要KWの順位推移を確認
- Ahrefs MCP `site-explorer-keywords-history` で過去12か月の同KWの順位・トラフィック推移を取得（季節性ベースライン）
- 競合サイトが同KWで順位上昇していないか確認（Ahrefs `site-explorer-organic-competitors`）
- **季節性の定量的分離**: GSCの前年同月インプレッションと比較し、「前年同月比 ±XX%」を算出。就活カレンダー（3-6月ES、6-9月面接、10-12月内定、1-2月就活準備）と照合
- 外部要因がある場合は「CTR変動のうち約X%ptは季節性/競合要因と推定」と定量的に分離

**6d. CV目標との乖離分析（CVファネル段階別）**:
- 達成率が80%未満の場合、ボトルネックを段階別に特定:
  1. **流入段階**: オーガニックセッション数が前月比で減少していないか → 減少なら記事リライトが優先
  2. **CTA段階**: セッション数は維持だがCTAクリック率が低下 → CTA配置・訴求改善が優先
  3. **フォーム段階**: CTAクリックはあるが `/materials/` PVとDLイベントに乖離 → フォームUX改善が優先
  - 各段階の数値を「セッション XX,XXX → CTAクリック XXX（クリック率 X.X%）→ DL完了 XX（完了率 XX%）」のファネル形式で表示
- 達成率が120%超の場合: 超過要因を特定（特定記事のバズ、季節イベント等）

### Step 7: ネクストアクション（具体的な実行内容を必ず示すこと）

**「タイトルを改善する」「CTAを最適化する」のような抽象的な提案は禁止。**
以下のレベルで具体化すること:

**7a. CTA改善提案**（CTAバナー変更がある場合）:
- 判定基準:
  - +5%以上: 改善 → 効果が出たデザインを他のどのページに展開するか、ページURLのリストを提示
  - -5%〜+5%: 変化なし → 次にテストすべきCTAコピーの具体案を2-3パターン提示（例:「無料で登録する」→「30秒で完了！今すぐ無料登録」）
  - -5%以下: 悪化 → ロールバック推奨 + 代替デザイン案の方向性を提示

**7b. CVR改善提案**:
- CV数が目標未達の場合、ボトルネックに応じた具体施策:
  - 流入が足りない場合 → セッション上位記事のうちCTAクリック率が低い記事を特定し、CTA配置変更案を提示
  - CTAクリックはあるがCV未達の場合 → フォーム/LPの改善ポイントを提示
- 各提案に期待効果の数値見積もりを付記（下記の計算式で算出）

**期待効果の計算式**:
```
期待クリック増分 = 月間インプレッション × (改善後CTR - 現在CTR)
改善後CTR = サイト過去90日の同順位帯平均CTR（取得不可の場合は業界目安: 1位28%, 2位15%, 3位11%, 4位8%, 5位7%, 6-10位3-5%）
期待CV増分 = 期待クリック増分 × 現在のCVR
```
例: 月間インプレッション 10,000 × (CTR 5%→8% = +3pt) = +300クリック/月。CVR 2%なら +6件/月。

**7c. 優先度とスケジュール**:
- P0（今週中）: 即座に実行すべき施策（ロールバック、緊急修正等）
- P1（来週中）: 計画的に実行すべき施策（新CTA案のA/Bテスト設計等）
- P2（今月中）: 中期的な改善施策

### Step 9: クライアント共有文（報告文書）生成

`Step 8` のCVレポートに基づき、クライアント送付用の報告文を作成する。  
**固定文構造を維持し、対象件数・効果値・方針のみ実データで置換すること。**

**固定テンプレート（骨子）**:
```markdown
お疲れ様です。
jo-katsu.com の記事{対象記事数}本についてリライト施策の期待効果を分析しましたので共有します。
今回のリライトで合計 {期待クリック増分}クリック回復（{増加率}） を想定しています。

下記の方向で進めても良いかご確認いただけますと幸いです。
---
対象記事（期待効果 Top5）

{Top5ブロック}

残りの記事については、以下のスプレッドシートにまとめております。
https://docs.google.com/spreadsheets/d/1O_eP4xUViDbgn0lXVbavyuy3H-0-Yt2H0WDmZakJ47M/edit?gid=642767266#gid=642767266
```

**Top5ブロックの形式**:
```markdown
https://jo-katsu.com/campus/XXXXX/
想定効果：+X,XXXクリック（現状X,XXX → X,XXX）
方針：[CVRボトルネック / CTA改善余地 / 検索意図に基づく1文]
```

**可変値の作成ルール**:
- `{対象記事数}`: CV改善対象として抽出した記事数
- `{期待クリック増分}`: CTA/CVR改善施策の期待クリック増分合計（算出可能な施策のみ）
- `{増加率}`: `(期待クリック増分 / 現状クリック合計) × 100`
- `方針`: `Step 7` の具体施策（P0/P1/P2）を1文に要約

**フォールバック**:
- クリック増分の直接算出が難しい場合は、`期待CV増分` か `主要KPI改善見込み` を使い、内訳数値を併記する

**出力ルール（必須）**:
- Markdown保存: `output/cv-report-client-brief-YYYY-MM-DD.md`
- ターミナル表示: 同内容をプレーンテキストで表示
- 共有文の根拠値は `Step 8` と整合させる

## 注意事項

- GA4イベントデータは通常翌日反映。当日分は含まれない可能性がある
- `pre_register_cta_click` と `dl_material_page_view` はGA4側で設定されたイベント名と完全一致が必要
- CTAバナー変更の記録は `/log-change` コマンドで事前に行う（`type: cta_banner` を指定）
- 目標値の更新: スプシ連携時は `scripts/sync_cv_targets_from_sheet.py` でスプシ→`config/cv-targets.json` を同期。未連携時はスプシの値を手動で `cv-targets.json` に転記

## 根拠明記ルール（必須）

レポート内の分析・判断・提案すべてに、以下のいずれかを必ず付記すること。根拠のない記述は禁止。

1. **定量データ**: どのツール（GSC / GA4 / Ahrefs / Clarity）のどの数値に基づく判断かを明記
   - 良い例: 「GSC compare_search_periods で CTR 2.1%→3.8%（+1.7pt）」
   - 悪い例: 「CTRが改善した」

2. **参照ソースURL**: 競合分析や施策提案の根拠となった外部ページのURLを明記
   - 良い例: 「参考: https://onecareer.jp/xxx （タイトル:『○○』、H2構成で比較表を含む）」
   - 悪い例: 「競合サイトを参考にタイトルを改善する」

**チェックリスト**（出力前に全項目を確認）:
- [ ] 原因分析の各項目に「※ データソース: [ツール名]」が記載されているか
- [ ] ネクストアクションの各提案に「根拠データ」と「参考URL」が記載されているか
- [ ] 「おそらく」「可能性がある」だけで終わる記述がないか（数値で裏付けること）
