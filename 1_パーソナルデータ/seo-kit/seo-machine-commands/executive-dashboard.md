# /executive-dashboard - 月次エグゼクティブダッシュボード

jo-katsu.com の月次SEO成果をクライアント報告用にサマリーする。
cv-report, organic-top100, weekly-report の結果を統合し、ROI判断・リソース配分の意思決定に必要な情報を集約する。

## 目的

- naimonoクライアントへの月次報告書として使用
- 施策のROI判断と次月の優先施策決定
- 予算配分・ライターリソース配分の根拠提供

## データソース

1. **output/cv-report-*.md**: 当月のCV指標レポート（最新版）
2. **output/organic-top100-*.md**: 当月のオーガニックTop100レポート（最新版）
3. **output/weekly-report-*.md**: 当月の週次レポート（全週分）
4. **config/cv-targets.json**: 月次目標値
5. **data/change_events/events.jsonl**: 施策実行ログ

## 実行手順

### Step 1: 月次レポート収集

- `output/` ディレクトリから当月の cv-report, organic-top100, weekly-report をすべて読み込む
- 各レポートの主要数値を抽出する

### Step 2: KPI達成度サマリー

以下のKPIについて、目標 vs 実績 vs 達成率を集計:

| KPI | 目標 | 実績 | 達成率 | 前月比 |
|-----|------|------|--------|--------|
| 会員登録数 | cv-targets.json | cv-reportから | % | % |
| 資料DL数 | cv-targets.json | cv-reportから | % | % |
| オーガニックセッション | - | GA4から | - | % |
| Top100合計セッション | - | organic-top100から | - | % |
| 要対応記事数 | 0件が理想 | organic-top100から | - | 前月との比較 |

### Step 3: 施策ROI分析

当月に実施した施策（`data/change_events/events.jsonl` から実行済みイベントを抽出）について:

| 施策 | 実施日 | 種類 | 期待効果（実施時の見積もり） | 実績効果 | 実現率 | 作業時間 | ROI |
|------|--------|------|--------------------------|---------|--------|---------|-----|
| evt-XXX | MM/DD | CTA変更 | +XXXクリック/月 | +YYYクリック/月 | ZZ% | Xh | YYY÷X |

- 効果測定が14日未満の施策は「測定中」と記載
- **実現率の集計**: 全施策の期待効果 vs 実績効果の平均実現率を算出
- 実現率が60%未満の場合、期待効果の見積もり精度に問題がある → 見積もり方法の見直しを提案

### Step 4: 次月の優先施策ランキング

organic-top100 と weekly-report のネクストアクションを統合し、以下の基準でランク付け:

```
総合スコア = 期待月間クリック増分 × CVR推定 ÷ 推定作業時間
```

| 優先度 | 対象ページ | 施策内容 | 期待クリック増 | 期待CV増 | 作業時間 | 総合スコア |
|--------|-----------|---------|-------------|---------|---------|-----------|
| 1 | [URL] | [施策] | +XXX/月 | +X/月 | Xh | XXX |
| 2 | [URL] | [施策] | +XXX/月 | +X/月 | Xh | XXX |

- 上位10件を推奨施策として提示
- 各施策に必要なリソース（ライター/開発者）を明記
- **月間合計の推定作業時間**と**推定リソースコスト**を算出

### Step 5: リスク・機会分析

**リスク**:
- 今月新たに🔴（要対応）になった記事数と影響セッション数
- 競合の注目すべき動き（新規コンテンツ、順位上昇等）を weekly-report から集約
- 季節性リスク: 来月の就活カレンダーイベントと予想される需要変動

**機会**:
- CTR改善余地のある記事（順位は良いがCTRが低い）のリスト
- 競合がカバーしていないキーワード群
- 季節性機会: 来月に需要増が見込まれるテーマ

### Step 6: 何もしないリスク

**現在の施策を停止した場合の影響予測**:
- 🔴記事が放置された場合の推定セッション損失: -XXX/月
- 競合が伸びているKW群を放置した場合の推定損失: -XXX/月
- 合計推定損失: -XXX セッション/月（≒ CV -X件/月）

## Step 7: クライアント共有文（報告文書）生成

上記ダッシュボード出力を要約し、クライアントへ即時送信できる文面を別途生成する。  
**文構造は固定し、数値・Top5・方針のみ当月データで置換する。**

**固定テンプレート（骨子）**:
```markdown
お疲れ様です。
jo-katsu.com の記事{対象記事数}本についてリライト施策の期待効果を分析しましたので共有します。
今回のリライトで合計 {期待クリック増分}クリック回復（{増加率}） を想定しています。

下記の方向で進めても良いかご確認いただけますと幸いです。
---
対象記事（期待効果 Top5）

{Top5ブロック}

残りの記事については、以下のスプレッドシートにまとめております。
https://docs.google.com/spreadsheets/d/1O_eP4xUViDbgn0lXVbavyuy3H-0-Yt2H0WDmZakJ47M/edit?gid=642767266#gid=642767266
```

**Top5ブロックの形式**:
```markdown
https://jo-katsu.com/campus/XXXXX/
想定効果：+X,XXXクリック（現状X,XXX → X,XXX）
方針：[KPI乖離 / ROI優先度 / 実行難易度を踏まえた1文]
```

**可変値の作成ルール**:
- `{対象記事数}`: 次月推奨施策ランキングの対象件数（通常Top10を起点に全体母数を併記）
- `{期待クリック増分}`: 次月推奨施策の期待クリック増分合計
- `{増加率}`: `(期待クリック増分 / 現状クリック合計) × 100`
- `方針`: 「次月の推奨施策 TOP10」の各施策理由を短文化して記載

**フォールバック**:
- クリック増分が未算出の施策は「期待CV増分」か「KPI改善見込み」に置換し、算出元レポートを明記する

**出力ルール（必須）**:
- Markdown保存: `output/executive-dashboard-client-brief-YYYY-MM-DD.md`
- ターミナル表示: 同内容をプレーンテキストで表示
- 共有文の数値は `cv-report` / `organic-top100` / `weekly-report` の引用値と一致させる

## 注意事項

- このレポートはクライアント報告にそのまま使用できるレベルの精度を目指す
- 数値はすべてソースレポートからの引用。独自の推計を加える場合は「推計」と明記
- 施策ROIの「実現率」は効果測定が完了している施策のみ対象
- 季節性の影響は前年同月データがある場合のみ記載

## 根拠明記ルール（必須）

レポート内の分析・判断・提案すべてに、以下のいずれかを必ず付記すること。根拠のない記述は禁止。

1. **定量データ**: どのソースレポート（cv-report / organic-top100 / weekly-report）のどの数値に基づく判断かを明記
   - 良い例: 「organic-top100 レポートより: /campus/26469 のセッション -42%（1,230→713）、主因はクエリ『ES 書き方』の順位下落（3位→7位）」
   - 悪い例: 「セッションが大幅に減少した」

2. **参照ソースURL**: 施策提案の根拠となったページURLを明記
   - 良い例: 「対象: https://jo-katsu.com/campus/26469 / 参考競合: https://onecareer.jp/xxx」
   - 悪い例: 「記事をリライトする」

**チェックリスト**（出力前に全項目を確認）:
- [ ] KPI達成度の各数値にソースレポート名が紐付いているか
- [ ] 次月の推奨施策TOP10の各施策に対象URL・根拠データ・参考URLが記載されているか
- [ ] リスク・機会の各項目に定量データが含まれているか
- [ ] 「おそらく」「可能性がある」だけで終わる記述がないか（数値で裏付けること）
