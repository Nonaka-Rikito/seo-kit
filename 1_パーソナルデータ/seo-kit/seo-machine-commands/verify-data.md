# /verify-data - データ検証

指定されたSEO分析データの正確性を3重チェックで検証する。

## パラメータ

`$ARGUMENTS` に検証対象を指定（省略時は直前の分析結果を検証）

## 実行手順

1. 検証対象データを特定（直前のコマンド出力 or 指定ファイル）
2. **1次検証: data_verifier.py**
   - 変化率の独立再計算
   - メトリクス値の範囲チェック（CTR 0-100%, 順位 1-200 等）
   - データ鮮度の確認（GSCの3日遅延を考慮）
   - サンプルサイズの統計的妥当性
   - クリック数 <= インプレッション数の整合性
3. **2次検証: cross_checker.py**
   - GSCクリック vs GA4セッションの照合
   - Ahrefs順位 vs GSC順位のクロスチェック
   - GA4トレンド vs GSCトレンドの方向性一致
   - Clarity vs GA4のセッション数照合
4. **3次検証: 信頼度スコア算出**
   - 全チェック結果から 0-100 スコアを計算
   - 失敗チェックの詳細を列挙

## 出力フォーマット

```
📋 データ検証レポート
━━━━━━━━━━━━━━━━━━

信頼度スコア: XX/100 (high / moderate / low)

## 1次検証（数値再計算）
[OK] 変化率: 実測+15.3% vs 主張+15.3%（一致）
[OK] CTR範囲: 2.4%（正常範囲内）
[WARNING] データ鮮度: 4日前（GSC通常遅延）
[OK] サンプルサイズ: 450セッション（十分）

## 2次検証（クロスチェック）
[OK] GSC vs GA4: 比率1.2x（正常）
[OK] 順位データ: Ahrefs 5位 vs GSC 5.3位（差0.3）
[OK] トレンド: 両方「増加」で一致

## 総合判定
✅ データは信頼できます / ⚠️ 一部注意が必要です
```
