# /organic-top100 - オーガニックセッション Top100 記事レポート

jo-katsu.com のオーガニックセッション上位100記事の月次比較レポートを生成する。
前月比（30日 vs 30日）で増減を分析し、要対応記事には原因分析と具体的な対策を提案する。

## データソース

1. **mcp-gsc**: ページ別検索パフォーマンス（クリック、インプレッション、順位、CTR）
2. **GA4 Python API**: ページ別オーガニックセッション数（`scripts/ga4_events.py`）
3. **Ahrefs MCP**: フラグ記事のキーワード順位・競合データ
4. **Clarity MCP**: フラグ記事のUX行動データ

## 実行手順

### Step 1: 期間計算

- GSCデータは3日遅延を考慮し、end_date = 3日前
- 今期: end_dateから30日間
- 前期: 今期の直前30日間
- 日付を `YYYY-MM-DD` 形式で算出しておく

### Step 2: GSCデータ取得（並行実行可）

**2a.** mcp-gsc の `get_advanced_search_analytics` で今期Top100を取得:
```
site_url: "https://jo-katsu.com/campus/"
dimensions: "page"
row_limit: 100
sort_by: "clicks"
sort_direction: "descending"
start_date: [今期開始]
end_date: [今期終了]
```

**2b.** mcp-gsc の `compare_search_periods` でページ別前期比較:
```
site_url: "https://jo-katsu.com/campus/"
dimensions: "page"
limit: 100
period1_start_date: [前期開始]
period1_end_date: [前期終了]
period2_start_date: [今期開始]
period2_end_date: [今期終了]
```

### Step 3: GA4オーガニックセッション取得

以下のPythonスクリプトを実行:
```bash
.venv/Scripts/python scripts/ga4_events.py organic-sessions-compare [今期開始] [今期終了] [前期開始] [前期終了] 100
```

JSONの `rows` 配列からページ別のセッション今期/前期/変化率を取得。

### Step 4: データ統合と整合性検証

GSCデータ（クリック、インプレッション、順位、CTR）とGA4データ（オーガニックセッション）をURLで結合。
URLの正規化に注意（末尾スラッシュ、パラメータ除去）。

**GA4 ↔ GSC 乖離検証**:
- URL別にGA4セッション数とGSCクリック数を比較
- 乖離が30%を超えるURLがあれば警告を記載（リファラスパム、パラメータ付きURL重複、キャッシュ等が原因の可能性）
- 乖離が大きいURLはフラグ判定の信頼度を下げる（信頼度スコアに反映）

### Step 5: フラグ記事の特定

以下の条件に該当する記事をフラグ:

**🔴 要対応**:
- オーガニックセッション前期比 **-30%以上** の下落
- **OR** GSC週次データ **4週連続下降**

**⚠️ 注意**:
- オーガニックセッション前期比 **-10%〜-30%** の下落

4週連続下降の確認方法（フラグ候補のみ実施）:
```
mcp-gsc get_advanced_search_analytics
  site_url: "https://jo-katsu.com/campus/"
  dimensions: "page,date"
  row_limit: 500
  start_date: 60日前
  end_date: 3日前
```
→ 対象ページでフィルタし、7日単位に集計して4週連続減少かチェック

### Step 6: フラグ記事の深掘り（🔴のみ）

各🔴記事に対して以下を実行:

**6a.** Ahrefs MCP `site-explorer-organic-keywords`:
- target: "jo-katsu.com"
- 対象ページの主要キーワードの順位変動を確認

**6b.** Clarity MCP `query-analytics-dashboard`:
- 対象ページのセッション数、スクロール深度、死にクリック等のUXデータ

**6c.** 原因分析（数値根拠を必ず示すこと。「おそらく」「可能性がある」だけの記述は禁止）:

以下の観点それぞれについて、データに基づく数値を示して分析する:

1. **クエリ別の順位・CTR変動**:
   - mcp-gsc `get_advanced_search_analytics` でページの主要クエリ（上位10件）の順位・CTR・クリックをBefore/Afterで比較
   - 「クエリ『○○ △△』: 順位 3.2→6.8（-3.6）、CTR 8.1%→3.2%（-4.9pt）、クリック -245」のように記載
   - どのクエリの下落が全体下落の何%を占めるかを算出
   - **検索意図の分類**: 各クエリを以下に分類し、対策の方向性を決定する:
     - `transactional`（登録・申込み意図）: 「就活サイト 登録」→ CTA・LP改善が優先
     - `commercial`（比較・検討意図）: 「就活エージェント おすすめ」→ ランキング構成・比較表の充実が優先
     - `informational`（情報収集意図）: 「ES 書き方」→ コンテンツ網羅性・鮮度が優先
     - 検索意図によってリライト方針が根本的に異なるため、必ず分類してから対策を立てる

2. **競合の順位変動とコンテンツ分析**:
   - Ahrefs MCP `site-explorer-organic-keywords` で主要KWのSERP変動を確認
   - 下落したクエリについて、代わりにランクアップした競合ページのURLを特定
   - 「クエリ『○○』で onecareer.jp/xxx が 8位→2位に上昇、当サイトは3位→7位に下落」のように記載
   - **競合ページのURLは必ず記載する**（後のリライト時に参照するため）
   - **競合コンテンツの具体分析**（🔴記事の主要競合1-2ページについて）:
     - 競合ページのタイトル: 「[競合のタイトル全文]」
     - 競合ページのH2見出し構成: 箇条書きで列挙
     - 当サイト記事との差分: 「競合は○○の観点を含むが当サイトは未カバー」「競合は△△形式（比較表/ステップ解説等）だが当サイトは□□形式」
     - 競合の優位点: 「なぜ競合が上位なのか」を1-2文で明記（情報の網羅性、構成の見やすさ、更新日の新しさ等）
   - Ahrefs MCP `site-explorer-broken-backlinks` で対象ページへの外部リンク喪失がないか確認（リンク喪失が原因の場合はリライトではなくリンク獲得施策が優先）

3. **検索需要の季節変動（定量ベースライン比較）**:
   - Ahrefs MCP `site-explorer-keywords-history` で主要KWの過去12か月の検索ボリューム推移を取得
   - GSCのインプレッション推移で需要変動を確認し、**前年同月比**を算出
   - 就活カレンダー（3-6月ES、6-9月面接、10-12月内定、1-2月就活準備）との一致を確認
   - 季節性が要因の場合「インプレッション XX,XXX→YY,YYY（-ZZ%）、前年同月も同様に-WW%で需要減が主因。順位は維持」と明記
   - **SERPフィーチャーの変動**: Ahrefs `serp-overview` で強調スニペット・PAA（People Also Ask）の出現状況を確認。「強調スニペットが新規出現し、通常の1位CTRが大幅に低下」等のケースを検出

4. **コンテンツの鮮度と品質**:
   - 最終更新日からの経過日数を確認
   - 記事内の年号や情報が古くなっていないか確認（例:「2025年版」のまま等）

**6e.** 現在のコンテンツ情報の取得（🔴記事について必ず実施）:
- 対象記事の**現在のタイトルタグ**を取得（GSCデータまたはページ読み込みから）
- 対象記事の**H2見出し一覧**を取得
- 対象記事の**メタディスクリプション**を取得
- これらはリライト提案の「現在の内容」欄に記載する（ライターが追加調査なしで作業開始できるようにする）

**6f.** 対策提案（リライト箇所を具体的に指示すること）:

**「タイトルを改善する」「内部リンクを強化する」のような抽象的な提案は禁止。**
以下のレベルで具体化すること:

各🔴記事について、P0/P1の優先度で以下を含む提案を作成:

- **リライト提案の場合**:
  - 変更箇所: タイトル/H2見出し/本文セクション/メタディスクリプション のどこを変えるか
  - 現在の内容: 「現タイトル: 『○○○○○』」
  - 変更案: 「変更案: 『△△△△△』」（具体的な文言を提示）
  - 変更理由: 「競合 [URL] では『□□□□□』という切り口で上位獲得。同様にXXの観点を追加」
  - 期待効果: 「順位 X位→Y位回復で月間 +ZZZ クリック相当」

- **コンテンツ追加の場合**:
  - 追加すべきセクションのH2/H3見出し案
  - 何を書くべきかの要点（200文字程度）
  - 参考にすべき競合記事のURL

- **内部リンク施策の場合**:
  - リンク元ページのURL（具体的に）
  - リンク先ページのURL（具体的に）
  - アンカーテキスト案

- **タイトル案の作成方針**: タイトル変更案を作る際は以下の観点から方針を明記する:
  - キーワード最適化（ターゲットKWの配置位置変更）
  - ベネフィット訴求（「〜のコツ」→「内定率XX%UPの〜」）
  - 数字・年号の追加（「2026年最新版」「XX選」）
  - 検索意図との一致（commercial意図なら「おすすめ」「比較」を含める等）
  - **必ず「なぜこのタイトル案にしたか」の理由を1文で説明する**

- **各提案に必ず付記**:
  - 参考URL: 競合記事やベンチマーク記事のURL（Ahrefsで特定したもの）
  - 推定作業時間: タイトル変更のみ=0.5h、セクション追加=2-4h、全面リライト=8-16h 等
  - 期待効果の数値見積もり（下記の計算式で算出）
  - **ROI指数 = 期待月間クリック増分 ÷ 推定作業時間**（この値が大きい順に実行を推奨）

**期待効果の計算式**:
```
期待クリック増分 = 月間インプレッション × (目標CTR - 現在CTR)
目標CTR = サイト過去90日の同順位帯平均CTR（GSCデータから算出。取得不可の場合は業界目安: 1位28%, 2位15%, 3位11%, 4位8%, 5位7%, 6-10位3-5%）
順位改善を見込む場合: 月間インプレッション × 目標順位のCTR - 現在クリック数
```
例: クエリ「ES 書き方」月間インプレッション 50,000、現在順位7位(CTR 3%)→目標3位(CTR 11%) = 50,000×(11%-3%) = +4,000クリック/月

### Step 8: クライアント共有文（報告文書）生成

`Step 7` で生成したレポートをもとに、クライアント共有用の報告文を作成する。  
**固定文の構造は維持し、数値と方針のみ分析結果で差し替えること。**

**固定テンプレート（骨子）**:
```markdown
お疲れ様です。
jo-katsu.com の記事{対象記事数}本についてリライト施策の期待効果を分析しましたので共有します。
今回のリライトで合計 {期待クリック増分}クリック回復（{増加率}） を想定しています。

下記の方向で進めても良いかご確認いただけますと幸いです。
---
対象記事（期待効果 Top5）

{Top5ブロック}

残りの記事については、以下のスプレッドシートにまとめております。
https://docs.google.com/spreadsheets/d/1O_eP4xUViDbgn0lXVbavyuy3H-0-Yt2H0WDmZakJ47M/edit?gid=642767266#gid=642767266
```

**Top5ブロックの形式**:
```markdown
https://jo-katsu.com/campus/XXXXX/
想定効果：+X,XXXクリック（現状X,XXX → X,XXX）
方針：[CTRギャップ / 順位 / 検索意図 / 構成課題に基づく1文]
```

**可変値の作成ルール**:
- `{対象記事数}`: このコマンドで分析対象にした記事数（通常100件、または条件適用後の対象件数）
- `{期待クリック増分}`: Top5ではなく、当該レポートで提示した「実行対象全体」の期待増分合計を優先
- `{増加率}`: `(期待クリック増分 / 現状クリック合計) × 100` を `%` 表記
- `方針`: 各記事の原因分析と推奨アクションから要約し、抽象表現を避ける

**フォールバック**:
- 期待クリック増分を算出できない場合は「主要KPI改善見込み」に置換し、根拠指標（セッション・CTR・順位）を併記する

**出力ルール（必須）**:
- Markdown保存: `output/organic-top100-client-brief-YYYY-MM-DD.md`
- ターミナル表示: 同内容を貼り付け可能なプレーンテキストで出力
- 本文中の数値・方針は、`Step 7` のレポート値と矛盾しないこと

## 注意事項

- GSCの `get_advanced_search_analytics` のrow_limitは最大25000なので100件は余裕
- GA4のオーガニックセッションとGSCクリックは定義が異なる（セッション≠クリック）。両方表示して補完する
- 日本語コンテンツの分析には `data_sources/japanese/` の `_ja` モジュールを使用
- レポートには必ずデータ信頼度を付加する（`data_sources/verification/`）

## 根拠明記ルール（必須）

レポート内の分析・判断・提案すべてに、以下のいずれかを必ず付記すること。根拠のない記述は禁止。

1. **定量データ**: どのツール（GSC / GA4 / Ahrefs / Clarity）のどの数値に基づく判断かを明記
   - 良い例: 「Ahrefs site-explorer-organic-keywords: クエリ『ES 書き方』で onecareer.jp/xxx が 8位→2位に上昇」
   - 悪い例: 「競合が順位を上げた」

2. **参照ソースURL**: 競合分析や施策提案の根拠となった外部ページのURLを明記
   - 良い例: 「参考: https://onecareer.jp/xxx （タイトル:『○○』、H2で比較表・体験談を含む点が優位）」
   - 悪い例: 「競合サイトを参考にリライトする」

**チェックリスト**（出力前に全項目を確認）:
- [ ] 原因分析の各テーブルに「※ データソース: [ツール名]」が記載されているか
- [ ] 競合コンテンツとの差分に競合ページの実URLが記載されているか
- [ ] 推奨アクションの各提案に「根拠」欄（参考URL + 参照データ）が記載されているか
- [ ] 「おそらく」「可能性がある」だけで終わる記述がないか（数値で裏付けること）
