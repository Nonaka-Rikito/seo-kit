# /weekly-report - 週次SEOレポート

jo-katsu.com の週次SEOパフォーマンスレポートを自動生成する。

## データソース

1. **mcp-gsc**: 検索パフォーマンス（クリック、インプレッション、順位、CTR）
2. **Ahrefs MCP**: 競合動向、キーワード順位データ
3. **Clarity蓄積データ**: UX指標のトレンド（data/clarity/）
4. **変更イベントログ**: 進行中の効果測定（data/change_events/）

## このレポートの位置づけ

**週次レポートはトレンド監視と早期警戒が目的。施策の実装判断は月次レポート（organic-top100）で行う。**
- 週次で検出した異変は「要観察」として記録し、月次で確認されたら施策化する
- 週次のP0施策は「明らかな異常（順位急落、エラー発生等）への緊急対応」のみ
- 通常の順位変動への対応はP1（来週確認）→月次で判断のフロー

## 実行手順

### Step 1: 期間計算

- GSCは3日遅延を考慮し、end_date = 3日前
- 今週: end_dateから7日間（月曜〜日曜を固定。例: レポート日2026-02-23の場合、今週=02-10〜02-16、先週=02-03〜02-09）
- 先週: 今週の直前7日間
- 先々週: 先週の直前7日間（トレンド確認用）
- **4週移動平均も算出**: 過去28日間の平均値を週次変動のノイズ除外に使用

### Step 1.5: 先週の推奨アクション進捗確認

前回の weekly-report（`output/weekly-report-*.md` の最新）を読み込み:
- 前回のP0施策が実行されたか確認（変更イベントログ `data/change_events/events.jsonl` と照合）
- 実行済み → 「完了」と記載し、効果の初期観測を記載
- 未実行 → 「未着手」と記載し、今回のレポートで優先度を再評価

### Step 2: サイト全体パフォーマンス

**2a.** mcp-gsc `get_performance_overview` で今週のサマリーを取得
**2b.** mcp-gsc `compare_search_periods` で今週 vs 先週を比較:
```
site_url: "https://jo-katsu.com/campus/"
dimensions: "page"
limit: 50
period1_start_date: [先週開始]
period1_end_date: [先週終了]
period2_start_date: [今週開始]
period2_end_date: [今週終了]
```

**2c.** mcp-gsc `get_advanced_search_analytics` でクエリ別の今週データ:
```
site_url: "https://jo-katsu.com/campus/"
dimensions: "query"
row_limit: 50
sort_by: "clicks"
sort_direction: "descending"
start_date: [今週開始]
end_date: [今週終了]
```

**2d.** GA4オーガニックセッション（GSCクリックとの整合性確認用）:
```bash
.venv/Scripts/python scripts/ga4_events.py organic-sessions [今週開始] [今週終了] 20
```

**2e.** サイト固有の位置別平均CTR算出（初回のみ。以降はキャッシュ利用）:
mcp-gsc `get_advanced_search_analytics` で過去90日の全クエリを取得し、順位帯別（1位, 2位, 3位, 4-5位, 6-10位, 11-20位）の平均CTRを算出。
→ Step 8b のCTR改善機会判定に使用（業界一般値ではなくサイト固有値を優先）

### Step 3: キーワード順位変動の深掘り分析

**3a.** mcp-gsc `compare_search_periods` でクエリ別の前週比較:
```
site_url: "https://jo-katsu.com/campus/"
dimensions: "query"
limit: 30
```

**3b.** 順位変動が大きい（±3位以上）クエリを特定

**3c.** 各変動クエリについて要因分析（**数値根拠を必ず示すこと**）:

- **順位上昇したクエリ**:
  - 何が寄与したか（コンテンツ更新、内部リンク追加、季節性等）を特定
  - 該当ページのCTR変化も確認（順位上昇でCTRも上がったか）

- **順位下落したクエリ**:
  - Ahrefs MCP `serp-overview` で該当クエリのSERP上位10件を確認
  - 代わりにランクアップした競合ページのURLを特定
  - 「クエリ『○○』: 当サイト 4位→8位、onecareer.jp/xxx が 9位→3位に上昇」のように記載
  - **競合ページのURLは必ず記載する**
  - **検索意図の分類**: 下落クエリを以下に分類し、対策方針を決定する:
    - `transactional`（登録・申込み意図）→ CTA・LP改善が優先
    - `commercial`（比較・検討意図）→ ランキング構成・比較表の充実が優先
    - `informational`（情報収集意図）→ コンテンツ網羅性・鮮度が優先
  - **SERPフィーチャー変動の確認**: Ahrefs `serp-overview` で、順位は維持なのにCTRが低下しているクエリについて:
    - 強調スニペットの新規出現（通常1位のCTRを大幅に低下させる）
    - PAA（People Also Ask）の追加
    - ナレッジパネルの出現
    - 「順位3位維持だがCTR 11%→5%に低下。強調スニペットが新規出現」のように記載

### Step 4: 競合動向分析

**4a.** Ahrefs MCP `site-explorer-organic-competitors` で jo-katsu.com の競合サイト動向を確認
**4b.** 主要競合（上位3サイト）のオーガニックトラフィック推移を確認
**4c.** 今週特に動きがあった競合がいれば:
- どのKW群で順位変動があったか
- 新規コンテンツが公開されていないか（Ahrefs `site-explorer-top-pages` で直近の新規ページを確認）

### Step 5: UX指標分析（Clarity）

**5a.** Clarity MCP `query-analytics-dashboard` で先週のUXサマリーを取得:
- セッション数、平均セッション時間、直帰率
- デッドクリック、レイジクリック、クイックバック

**5b.** 前週比で悪化した指標があれば:
- どのページで悪化しているかを特定
- 「/campus/xxx でレイジクリック +XX%。該当要素を確認する必要あり」のように具体化

### Step 6: 変更イベント確認

`data/change_events/events.jsonl` から測定中のイベントを確認:
- 変更日から14日経過 → 効果測定結果のサマリーを記載
- 14日未満 → 経過日数と中間観測を記載

### Step 7: データ検証

`data_sources/verification/` で全数値を検証し、信頼度スコアを付加。

### Step 8: ネクストアクション作成

**「○○を改善する」「△△を強化する」のような抽象的な提案は禁止。**
以下のレベルで具体化すること:

**8a. 順位下落への対応**:
今週順位が下落したクエリに対して:
- リライト対象の記事URL
- **現在のタイトル全文・H2見出し一覧・メタディスクリプション**を記載（ライターが追加調査なしで着手できるようにする）
- 変更箇所（タイトル/見出し/本文セクション/メタディスクリプション）
- 現在の内容と変更案の具体的な文言 + **変更理由を1文で説明**
- 参考にすべき競合記事のURL + **競合ページのタイトル・見出し構成の何が優れているか**
- 推定作業時間: タイトル変更のみ=0.5h、セクション追加=2-4h、全面リライト=8-16h
- 期待効果（計算式付き）:「月間インプレッション XX,XXX × (CTR X%→Y%) = +ZZZ クリック/月」
- **ROI指数 = 期待月間クリック増分 ÷ 推定作業時間**
- **4週移動平均との比較**: 「今週の下落が一時的ノイズか、トレンド的な下落か」を判定。4週平均も下落傾向なら月次での対応を推奨、今週のみの急落なら来週まで様子見

**8b. CTR改善機会**:
順位は良い（10位以内）がCTRが低い（サイト固有の期待CTRを下回る）クエリを特定:
※期待CTR = Step 2eで算出したサイト固有の位置別平均CTR（未算出の場合のフォールバック: 1位28%, 2位15%, 3位11%, 4位8%, 5位7%, 6-10位3-5%）
- 対象クエリとページURL
- **現在のタイトル全文**: 「[実際のタイトルタグ]」
- **現在のメタディスクリプション全文**: 「[実際のdescription]」
- CTR改善のためのタイトル変更案（具体的な文言）+ **変更理由を1文で説明**
- 期待効果（計算式付き）:「月間インプレッション XX,XXX × (CTR X%→Y% = +Z.Zpt) = +ZZZ クリック/月」
- 推定作業時間: X時間

**8c. 競合対策**:
今週競合が伸びたKW群に対して:
- 当サイトの該当ページURL（既存記事がある場合）
- 競合ページのURL
- **競合コンテンツの具体分析**:
  - 競合ページのタイトル: 「[タイトル全文]」
  - 競合ページのH2見出し構成（箇条書きで列挙）
  - 当サイト記事との差分: 「競合は○○の観点を含むが当サイトは未カバー」「競合は△△形式だが当サイトは□□形式」
  - 競合の優位点を1-2文で明記
- 新規記事が必要な場合: タイトル案、ターゲットKW、推定月間検索ボリューム、競合記事の文字数目安
- 推定作業時間とROI指数

**8d. UX改善**:
Clarityで問題が検出されたページに対して:
- 具体的なページURLと検出された問題
- 改善案（例:「CTAボタンの位置を○○の直下に移動」「フォームのステップ数をX→Yに削減」）

### Step 10: クライアント共有文（報告文書）生成

`Step 9` の週次レポートから、クライアントへそのまま送付できる共有文を作成する。  
**文構造は固定し、今週の実数値とTop5対象のみ差し替えること。**

**固定テンプレート（骨子）**:
```markdown
お疲れ様です。
jo-katsu.com の記事{対象記事数}本についてリライト施策の期待効果を分析しましたので共有します。
今回のリライトで合計 {期待クリック増分}クリック回復（{増加率}） を想定しています。

下記の方向で進めても良いかご確認いただけますと幸いです。
---
対象記事（期待効果 Top5）

{Top5ブロック}

残りの記事については、以下のスプレッドシートにまとめております。
https://docs.google.com/spreadsheets/d/1O_eP4xUViDbgn0lXVbavyuy3H-0-Yt2H0WDmZakJ47M/edit?gid=642767266#gid=642767266
```

**Top5ブロックの形式**:
```markdown
https://jo-katsu.com/campus/XXXXX/
想定効果：+X,XXXクリック（現状X,XXX → X,XXX）
方針：[今週の順位変動 / CTR差分 / 検索意図 / 競合差分に基づく1文]
```

**可変値の作成ルール**:
- `{対象記事数}`: 当週で改善候補として扱った記事数
- `{期待クリック増分}`: P0/P1/P2のうち実行想定施策の期待増分合計（重複除外）
- `{増加率}`: `(期待クリック増分 / 現状クリック合計) × 100`
- `方針`: `Step 8` の推奨アクションを要約し、具体文言を残す

**フォールバック**:
- 期待クリック増分を出せない場合は、`今週の主要KPI改善見込み` に置き換え、根拠数値（クリック、CTR、順位）を併記する

**出力ルール（必須）**:
- Markdown保存: `output/weekly-report-client-brief-YYYY-MM-DD.md`
- ターミナル表示: 同内容をプレーンテキストで表示
- 共有文の数値は `Step 9` の出力値と整合させる

## 注意事項

- GSCデータは3日遅延が正常。4日以上遅延の場合は警告を記載
- 就活カレンダーの季節性を必ず考慮（3-6月ES、6-9月面接、10-12月内定、1-2月就活準備）
- ネクストアクションは必ず数値根拠と競合URLを添えること
- レポートには必ずデータ信頼度を付加する（`data_sources/verification/`）

## 根拠明記ルール（必須）

レポート内の分析・判断・提案すべてに、以下のいずれかを必ず付記すること。根拠のない記述は禁止。

1. **定量データ**: どのツール（GSC / GA4 / Ahrefs / Clarity）のどの数値に基づく判断かを明記
   - 良い例: 「GSC compare_search_periods でクエリ『ES 書き方』順位 3.2→6.8、CTR 8.1%→3.2%」
   - 悪い例: 「順位が下落した」

2. **参照ソースURL**: 競合分析や施策提案の根拠となった外部ページのURLを明記
   - 良い例: 「参考: https://onecareer.jp/xxx （タイトル:『○○』、H2構成で比較表を含む）」
   - 悪い例: 「競合サイトを参考にタイトルを改善する」

**チェックリスト**（出力前に全項目を確認）:
- [ ] 順位変動の要因分析に「※ データソース: [ツール名]」が記載されているか
- [ ] 下落クエリに対応する競合ページのURLが記載されているか
- [ ] 推奨アクションの各提案に「根拠データ」と「参考URL」が記載されているか
- [ ] 「おそらく」「可能性がある」だけで終わる記述がないか（数値で裏付けること）
