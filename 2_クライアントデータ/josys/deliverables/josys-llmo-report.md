# Josys AI検索最適化（LLMO）施策提案書

**提出日**: 2026年3月3日
**作成**: malna株式会社
**宛先**: Josys マーケティングチーム / Global チーム

---

## 1. 本提案の背景

B2B購買プロセスにおけるAI活用が急速に進んでいます。

- B2Bバイヤーの **87%** が導入検討時にAIツールを利用（2025年調査）
- LLM経由のサインアップは、Google検索経由と比較して **コンバージョン率6倍**（Webflow社実績）
- FAQPageスキーマを実装したページは、未実装ページと比較して **AI引用率2.7倍**（Relixir社調査）

こうした状況の中、SaaS管理カテゴリの主要プレイヤー7社（Zylo、Productiv、BetterCloud、Zluri、Torii、LeanIX、Snow Software）を調査した結果、**AI検索向けの専用ページ（brand-facts、llms.txt等）を整備している企業はゼロ**であることが判明しました。

つまり、SaaS管理市場全体がLLMO（LLM最適化）において空白地帯であり、Josysが先行実装することで**ファーストムーバー・アドバンテージ**を確立できる機会があります。

## 2. Josysサイトの現状と課題

### 2.1 AI検索対応状況の診断結果

現在のjosys.comをAI検索の観点で評価した結果を以下にまとめます。

| 項目 | 現状 | 課題 |
|------|------|------|
| AI向けファクトシート（brand-facts） | 未作成 | AIが「Josysとは？」に回答する際の一次ソースが存在しない |
| 競合比較ページ | article-hub内に1件のみ | 「Josys vs ○○」の比較クエリで引用される機会を逸失 |
| 総合FAQページ | 独立ページなし | FAQ形式はAI引用率が高いが未活用 |
| AI向けサイトガイド（llms.txt） | 未作成 | AIクローラーがサイト構造を理解する手段がない |
| 構造化データ（スキーマ） | トップページのFAQPageのみ | 大半のページで機械可読性が低い |
| 事例ページ | 英日12社以上が存在 | スキーマ未実装のためAIが数値を抽出しづらい |
| 連携アプリページ | 350以上のアプリを掲載 | 「350+」という強みが構造化されていない |
| 料金ページ | 見積もり形式で存在 | FAQ・スキーマなし |

### 2.2 競合7社のLLMO対応状況

Josysの主要競合について、AI検索最適化の実装状況を調査しました。

| 競合 | brand-facts | 競合比較ページ | llms.txt | スキーマ実装 |
|------|:-----------:|:-------------:|:--------:|:-----------:|
| Zylo | × | 2ページ | × | 基本のみ |
| Productiv | × | 3ページ | × | 基本のみ |
| BetterCloud | × | 5ページ以上（最多） | × | 基本のみ |
| Zluri | × | 多数（ブログ形式含む） | × | 動的JS生成 |
| Torii | × | 2ページ | × | 確認不可 |
| LeanIX (SAP) | × | × | × | HubDB連携 |
| Snow (Flexera) | × | × | × | 確認不可 |
| **Josys** | **×** | **1件のみ** | **×** | **最小限** |

**結論**: 全競合がllms.txt・brand-factsともに未実装。比較ページはBetterCloudが最も積極的（5ページ以上）で、Zluri・Productivも展開中。Josysは比較コンテンツにおいても後れを取っている状態です。

## 3. 施策提案

AI検索でJosysが適切に引用・推薦されるために、以下の施策を3つの優先度に分けて提案します。

### 3.1 優先度A：必須ページの新規作成

AI引用の核となるページ群です。これらがなければ、AIが正確にJosysを推薦するための情報基盤が不足します。

#### (1) llms.txt — AI向けサイトガイド

| 項目 | 内容 |
|------|------|
| 概要 | AIクローラーにサイト構造と重要ページを伝えるガイドファイル |
| 設置場所 | `josys.com/llms.txt`（英語）/ `josys.com/llms-ja.txt`（日本語） |
| 内容 | 企業概要、主要プロダクトページ、機能ページ、ソリューション、リソースへのリンク一覧 |
| 想定工数 | 約3時間（英日合計） |

llms.txtは近年AIクローラー向けに普及しつつある標準フォーマットで、競合ではTorii、BetterCloudが関連ファイルの実装を始めています。Josysが早期に設置することで、AI検索エンジンからの優先的なクロールが期待できます。

#### (2) /brand-facts — AIファクトシート

| 項目 | 内容 |
|------|------|
| 概要 | 「Josysとは？」「Josysの特徴は？」にAIが回答する際の一次資料 |
| 設置場所 | `josys.com/brand-facts`（英語）/ `josys.com/jp/brand-facts`（日本語） |
| 掲載項目 | 企業概要 / 基本情報テーブル（設立・本社・親会社・顧客数・連携アプリ数・認証） / 製品カテゴリ / 主要機能 / ターゲット顧客 / 受賞歴 / FAQ |
| スキーマ | Organization + FAQPage + SoftwareApplication（JSON-LD） |
| 想定工数 | 約8時間（英日合計） |

AIは複数の情報源をクロスチェックして回答を生成します。brand-factsページは、外部メディアの記事内容を「裏付ける」公式の一次ソースとして機能し、AI推薦の確度を高めます。

#### (3) /compare/josys-vs-[競合名] — 競合比較ページ（5ページ）

「Josys vs ○○」「○○ alternative」といったAI検索クエリで引用されるためのページです。

**作成対象（優先順）**:

| 順位 | ページ | 選定理由 |
|:----:|--------|----------|
| 1 | josys-vs-zylo | SMP市場最大手。比較検索ボリュームが最も大きい |
| 2 | josys-vs-productiv | 直接競合。エンタープライズ層で顧客が重複 |
| 3 | josys-vs-bettercloud | 比較ページを最も多く展開している競合。防衛的に必要 |
| 4 | josys-vs-zluri | 急成長中。攻めの比較コンテンツを多数展開 |
| 5 | josys-vs-torii | 中堅競合。他社が既に比較ページを作成済み |

**各ページの構成方針**:

1. 冒頭50語で結論を提示（Answer-First原則）
2. HTML比較テーブルで機能・価格・サポートを対比（画像ではなくテキストテーブルを使用）
3. Josysの強み（3〜5点）を具体的に記載
4. 競合の強み（2〜3点）も記載し、客観性を担保
5. 「こんな企業にはJosysがおすすめ」のユースケース別推薦
6. FAQ（3〜5問）をFAQPageスキーマ付きで実装
7. 最終更新日を必ず表示（更新30日以内のコンテンツはAI引用率3.2倍）

**想定工数**: 各ページ4〜6時間 × 5ページ = 20〜30時間

#### (4) /faq — 総合FAQページ

| 項目 | 内容 |
|------|------|
| 概要 | FAQPageスキーマ実装ページはAI引用率2.7倍。Josysには現在独立FAQページがない |
| 設置場所 | `josys.com/faq`（英語）/ `josys.com/jp/faq`（日本語） |
| カテゴリ | 製品 / 料金 / セキュリティ / 導入 / 連携 / 競合比較 |
| 回答の長さ | 各40〜60語（AI抽出に最適なサイズ） |
| スキーマ | FAQPage（必須） |
| 想定工数 | 約8時間（英日合計） |

### 3.2 優先度B：既存ページの強化

既にコンテンツが存在するページに対して、構造化データの追加やFAQの追加を行うことで、AI引用率を向上させます。

| 対象ページ | 改善内容 | 想定工数 |
|-----------|---------|:--------:|
| /case-studies | Article / CaseStudyスキーマの追加。ROI数値（コスト削減額、時間短縮率）を構造化データとして明示 | 4時間 |
| /integrations | SoftwareApplication + ItemListスキーマの追加。「350+」の数値を機械可読化 | 4時間 |
| /pricing | 料金に関するFAQ（5〜8問）の追加。FAQPageスキーマ実装 | 4時間 |

### 3.3 優先度C：技術基盤整備

ページ作成と並行して、サイト全体の技術基盤をAI検索に対応させます。

| 施策 | 内容 | 想定工数 |
|------|------|:--------:|
| robots.txt更新 | AI検索ボット（ChatGPT-User、PerplexityBot、Claude-Web等）へのアクセス許可を追加 | 30分 |
| Organizationスキーマ統一 | 全ページに統一的なJSON-LD（企業情報・親会社・G2評価等）を実装 | 2時間 |
| 全ページに最終更新日を表示 | 更新30日以内のコンテンツはAI引用率3.2倍の効果がある | 2時間 |
| 外部プロフィールの同期 | G2・LinkedIn・Crunchbase・Wikidataの企業情報をサイトと完全一致させる | 2時間 |

> **補足**: robots.txtにおける `GPTBot`（学習用クローラー）と `ChatGPT-User`（検索・引用用クローラー）は別物です。AI検索での引用可視性を確保するには、後者を許可する必要があります。

## 6. 外部メディアとの連携構造

本施策は、自社サイト単独で完結するものではありません。外部メディア施策と組み合わせることで、AIの推薦確率が相乗的に高まります。

```
外部メディア                          自社サイト（josys.com）
（note, ITmedia 等）

┌──────────────────────┐          ┌──────────────────────┐
│                      │          │                      │
│ 役割：推薦する        │────────→│ 役割：裏付ける         │
│                      │          │                      │
│ ・比較記事           │          │ ・/brand-facts       │
│ ・導入事例インタビュー │          │ ・/compare/josys-vs-* │
│ ・業界トレンド解説    │          │ ・/case-studies       │
│                      │          │ ・/faq               │
│                      │          │ ・llms.txt           │
└──────────────────────┘          └──────────────────────┘
         │                                  │
         └──────────┐    ┌─────────────────┘
                    ↓    ↓
        AIが複数ソースをクロスチェック
              ↓
        情報が一致 → Josys推薦の確度UP
```

AIは回答生成時に複数の情報源を照合します。外部メディアが「Josysがおすすめ」と発信し、自社サイトがその根拠（事実データ・比較情報・FAQ）を構造化データ付きで提供している状態を作ることが、LLMO施策の最終ゴールです。
