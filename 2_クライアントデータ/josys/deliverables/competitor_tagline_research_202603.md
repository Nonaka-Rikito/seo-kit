### 空白領域・差別化機会

**発見1: 「経営層 × 課題訴求」のポジションが空白**

ほぼすべての競合が**情シス担当者向け**にメッセージを設計している。経営層（CxO）に直接刺さる課題訴求を打っている競合は、SaaS管理・ID管理領域にはいない。Sansanが経営層寄りだが、領域が異なる。

→ **Josysのたたき台B案（経営の説明責任系）は、競合が誰もいない空白ポジションを狙えている**

**発見2: 「規模の壁」を訴求している競合がいない**

「SaaS 300本」「従業員数千人」というエンタープライズ特有の課題を正面から言語化している競合がいない。AdminaもHENNGEも中堅〜大手を対象としているが、**規模の痛み**をコピーに落とし込んではいない。

→ **Josysのたたき台C案（規模×自動統制系）も独自性がある**

**発見3: 「自動化」は差別化にならない**

Admina「アカウント管理の自動化」、HENNGE「オールインワン」、各社とも「自動化」「一元管理」は使い尽くされたワード。ここで差別化するのは難しい。

## 3. 現行タグライン案の評価

### 案A（問いかけ系）の評価

| 案 | 強み | 弱み |
|----|------|------|
| A-1「記憶と手作業に頼っていませんか」 | 現場の痛みが具体的。情シスに刺さる | 競合（Admina等）と訴求が重なりやすい。経営層には響きにくい |
| A-2「仕組みで回せていますか」 | 「仕組み」という上位概念で差別化できる | やや抽象的。3秒で伝わるかギリギリ |
| A-3「規模に耐えられない」 | エンタープライズの痛みを直撃。競合が言っていない | 断定型で力強い。ただし「人に依存した」の前提が伝わるか要検証 |

### 案B（経営の説明責任系）の評価

| 案 | 強み | 弱み |
|----|------|------|
| B-1「誰が何をどこまで使えるか、即答できますか」 | **最も差別化されている**。競合が完全に空白のポジション。経営者のペインを直撃 | 展示会の3秒ルールではやや長い |
| B-2「取締役会で報告できる状態ですか」 | 「取締役会」で一発で経営層ターゲットだと分かる | CIO/CFO以外の来場者には自分事化しにくい |
| B-3「見えていないものは統制できない」 | 普遍的な真理で納得感が高い | Admina「可視化」と表現が被る |

### 案C（規模×自動統制系）の評価

| 案 | 強み | 弱み |
|----|------|------|
| C-1「増え続けるSaaS。統制の仕組み」 | 課題の拡大感が伝わる | 「増え続ける」はSaaS管理各社が使いがちなフレーズ |
| C-2「SaaS300本の権限管理を、ルールで自動化」 | 数字の具体性が強い。エンタープライズ感が出る | 「自動化」がAdminaと被る |
| C-3「部門ごとにバラバラの管理を、全社のルールで統一」 | 「バラバラ→統一」の構造が明快 | やや説明的。タグラインとしてはインパクト不足 |

### 組み合わせ案D の評価

| 案 | 強み | 弱み |
|----|------|------|
| D-1「即答できますか＋ルールで自動統制」 | メイン+サブのバランスが良い | メインが長め |
| D-2「まだ人手で回していますか＋仕組みで動かす」 | 「人手→仕組み」の対比が分かりやすい | 「人手で回す」はやや情シス向きで経営層に弱い |
| D-3「見えていないSaaSは統制できない＋可視化→自動制御」 | 論理的で説得力がある | Admina「可視化でリスク削減」と完全に被る |

### 総合評価

- **最も競合と差別化されている**: B-1、B-2（経営の説明責任系）
- **最も被りリスクが高い**: A-1（Adminaと重複）、B-3・D-3（「可視化」がAdminaと重複）
- **表現の強度が最も高い**: A-3（断定型）、B-1（問いかけ型で切迫感）

## 4. 改善タグライン案

調査結果を踏まえ、以下の方針で改善案を策定する。

**方針**:
1. 「経営層 × 課題訴求」の空白ポジションを確保する（最大の差別化機会）
2. 「規模の壁」という競合が未使用の訴求軸を活用する
3. 「可視化」「自動化」「一元管理」等の陳腐化ワードを避ける
4. カタカナ抽象語（ガバナンス、アイデンティティ）は使わない
5. 3秒で伝わる端的さを維持する

---

### 改善案1: 「経営の死角」型（B-1の改善）

| 項目 | 内容 |
|------|------|
| **メインコピー** | **誰が、何を、どこまで使えるか。即答できますか。** |
| **サブコピー** | 全社のSaaS利用を、ルールで自動統制するプラットフォーム。 |
| **差別化ポイント** | 競合が誰も「経営者への問い」を投げていない。このポジションは完全に空白。「誰が・何を・どこまで」は権限管理の本質を経営言語で表現 |
| **想定ターゲット** | CIO / CFO / 情シス責任者 |
| **訴求マップ位置** | 経営層 × 課題訴求（空白ポジション） |

※たたき台B-1/D-1からの改善点: 句読点で区切りリズムを改善。サブコピーで「ルールで自動統制」というJosys独自の強みを端的に提示。

---

### 改善案2: 「規模の限界」型（A-3の強化）

| 項目 | 内容 |
|------|------|
| **メインコピー** | **SaaS 300本。もう人手では管理しきれない。** |
| **サブコピー** | 増え続けるアプリと権限を、全社のルールで自動制御。 |
| **差別化ポイント** | 「300本」という数字がエンタープライズの来場者に刺さる。中小向けの競合（Admina等）には使えないコピー。「管理しきれない」は体感的な痛み |
| **想定ターゲット** | 情シス責任者 / DX推進担当 |
| **訴求マップ位置** | 情シス〜中間 × 課題訴求 |

---

### 改善案3: 「統制不能」型（B-3の改善・Admina被り回避）

| 項目 | 内容 |
|------|------|
| **メインコピー** | **把握できていないSaaSが、最大のリスクになる。** |
| **サブコピー** | 利用実態の把握から権限の自動統制まで。ルールで動くIT管理を。 |
| **差別化ポイント** | 「見えていない→統制できない」をさらに踏み込み「リスクになる」と経営インパクトに変換。Adminaの「リスク削減」は手段訴求だが、本案はリスクの存在自体を突きつける |
| **想定ターゲット** | CIO / CISO / 情シス責任者 |
| **訴求マップ位置** | 経営層寄り × 課題訴求 |

---

### 改善案4: 「属人化の終わり」宣言型

| 項目 | 内容 |
|------|------|
| **メインコピー** | **IT管理を、人の記憶から、会社のルールへ。** |
| **サブコピー** | SaaSの利用・権限・コストを、ポリシーで自動統制するプラットフォーム。 |
| **差別化ポイント** | 「人の記憶→会社のルール」という対比が分かりやすく、属人化の課題を一言で伝える。A-1の問いかけ型を宣言型に転換し、インパクトを強化 |
| **想定ターゲット** | 情シス責任者 / 経営層 |
| **訴求マップ位置** | 中間 × 課題訴求寄り |

---

### 改善案5: 「経営報告」型（B-2の改善）

| 項目 | 内容 |
|------|------|
| **メインコピー** | **全社のIT利用状況、経営会議で報告できますか。** |
| **サブコピー** | SaaSの利用・権限・コストを即座に把握し、ルールで自動統制。 |
| **差別化ポイント** | 「取締役会」より「経営会議」の方が幅広い経営層に自分事化される。「報告できるか」という問いが説明責任を突く |
| **想定ターゲット** | CFO / CIO / 経営企画 |
| **訴求マップ位置** | 経営層 × 課題訴求 |

---

### 改善案6: 「ルールで動く」宣言型

| 項目 | 内容 |
|------|------|
| **メインコピー** | **SaaS管理は、ルールで動く時代へ。** |

## 5. 推奨案とその理由

### 第1推奨: 改善案1「経営の死角」型

> **誰が、何を、どこまで使えるか。即答できますか。**
> 全社のSaaS利用を、ルールで自動統制するプラットフォーム。

**推奨理由**:
1. **競合との完全差別化**: SaaS管理・ID管理領域で「経営者への問い」を打っている企業はゼロ。HENNGE・Adminaは情シス現場向け、SmartHRは人事向け。経営層に直接語りかけるこのポジションは完全に空白
2. **Josys独自の強み反映**: 「誰が・何を・どこまで」はJosysが目指すPolicy-Based Identity Governanceの本質を、カタカナ語を使わず日本語で表現している
3. **展示会での効果**: 問いかけ型は来場者を立ち止まらせる力がある。CxOが通りがかりに「うちは即答できるか？」と自問する仕掛けになる
4. **たたき台D-1をベースにしており、社内検討の方向性と整合**

---

### 第2推奨: 改善案4「属人化の終わり」宣言型

> **IT管理を、人の記憶から、会社のルールへ。**
> SaaSの利用・権限・コストを、ポリシーで自動統制するプラットフォーム。

**推奨理由**:
1. **ビフォー→アフターの構造が明快**: 「人の記憶→会社のルール」は課題と解決を一文で両立する効率的なコピー
2. **情シスにも経営層にも刺さる**: 情シスは「属人化の脱却」、経営層は「仕組み化」として読める二重構造
3. **記憶に残りやすい**: 対比構造は壁面で3秒で読める。「人→会社」「記憶→ルール」の二重対比がリズムを生む
4. **競合と被らない**: HENNGE「守る」、Admina「削減」とは完全に異なる軸

---

### 第3推奨: 改善案8「数字で突く」型

> **御社のSaaS、何本ですか。即答できない、が答えです。**
> 全社のSaaS利用を把握し、権限をルールで自動統制。

**推奨理由**:
1. **展示会ブースでの停止力が最も高い**: 問いかけ+皮肉の構造は通行者を立ち止まらせる力がある
2. **エンタープライズの痛みを直撃**: 大企業ほど「何本使っているか分からない」という実感がある
3. **ただしリスクもある**: トーンがやや挑発的。ブランドイメージとの整合性は要検討

---

### 推奨の組み合わせパターン

展示会ブースでは複数面に異なるコピーを配置できるため、以下の組み合わせも有効。

| 配置 | コピー | 狙い |
|------|--------|------|
| **正面壁面（メイン）** | 改善案1「誰が、何を、どこまで使えるか。即答できますか。」 | 経営層の足止め |
| **側面パネル左** | 改善案4「IT管理を、人の記憶から、会社のルールへ。」 | 情シス責任者への共感 |
| **側面パネル右** | 改善案2「SaaS 300本。もう人手では管理しきれない。」 | エンタープライズ規模の実感 |
