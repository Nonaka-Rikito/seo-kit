## エグゼクティブサマリー

### 背景

jo-katsu.com は月間約16万セッション（前年比 **-64.9%**、約30万セッションの損失）という深刻なトラフィック減少に直面している。Google AI Overview の台頭により、従来型の長文解説記事のCTRが激減しており、記事リライト施策と並行して**バナー/CTA クリエイティブの差替え効果を正確に計測する仕組み**の構築が急務である。

### 現状資産

- **GAS 計測システム**: コード完成済み（7ファイル）、**未デプロイ**
- **GA4 / GSC API**: Node.js CLI スクリプト利用可能、サービスアカウント鍵発行済み
- **バナー素材**: `cta-banners/` に MP4 6件格納済み
- **Config.gs**: プレースホルダ状態（設定値未入力）

### 3領域統合の核心

| 領域 | 担当 | 核心提案 |
|------|------|---------|
| **運用効率化** | efficiency-specialist | 手動作業30秒/件、自動化率92%、月額0円 |
| **データ計測設計** | data-measurement-expert | DID分析による因果推論、対照群設計、バイアス排除 |
| **マーケット分析** | market-analyst | 階層化KPI体系（Tier 1/2/3）、統計検定、外部要因排除 |

### 最優先アクション（P0）

1. **GSC 閲覧権限の即時取得** -- CTR計測の前提条件（全3提案で指摘）
2. **GAS デプロイ** -- Config.gs に4値を設定するだけで計測開始可能
3. **対照群URLの選定** -- DID分析の基盤（data/market両提案で最重要と位置付け）

## 2. 統合アプローチ: データ計測設計

> 出典: data-measurement-design.md

### 2.1 設計原則

1. **因果推論の担保**: DID（差分の差分）分析により外部要因を分離
2. **データ品質第一**: 欠損・遅延・定義のブレを前提とした設計
3. **バイアスの明示**: 完全排除は不可能。残存バイアスを常に記録

### 2.2 計測指標（全16項目）

| # | カテゴリ | 指標 | データソース |
|---|---------|------|-------------|
| 1 | 流入量 | セッション数 | GA4 |
| 2 | 流入量 | PV数 | GA4 |
| 3 | エンゲージメント | エンゲージセッション数 | GA4 |
| 4 | エンゲージメント | エンゲージメント率 | GA4 |
| 5 | エンゲージメント | 平均セッション時間 | GA4 |
| 6 | エンゲージメント | 直帰率 | GA4 |
| 7 | コンバージョン | CV数 | GA4 |
| 8-11 | 読了 | スクロール 25%/50%/75%/90% 到達数 | GA4 (GTM) |
| 12 | 検索 | クリック数 | GSC |
| 13 | 検索 | 表示回数 | GSC |
| 14 | 検索 | CTR | GSC |
| 15 | 検索 | 平均掲載順位 | GSC |
| 16 | 画像検索 | 画像クリック数 + 画像表示回数 | GSC |

### 2.3 Before/After 比較設計

```
  変更前14日間          変更日      GSCバッファ3日    変更後14日間
|<--- Before --->|      |      |<--- Buffer --->|<--- After --->|
                        ^                                        ^
                    バナー差替え                            レポート生成
                                                         (変更日+17日)
```

**計測期間のガイドライン（market-analyst 提案との統合）:**

| 記事カテゴリ | 日次セッション | Before | After | 判定方式 |
|------------|--------------|--------|-------|---------|
| 大規模（MBTI一覧等） | 50+/日 | 14日（Phase 1）/ 28日（Phase 2+） | 14日 + 28日速報 | 個別評価可 |
| 中規模 | 15-50/日 | 14日 / 28日 | 14日 + 42日精査 | 個別評価可 |
| 小規模（Vol 500-2k） | 5-15/日 | 14日 / 28日 | 14日 + 56日精査 | クラスター評価 |
| 鮮度更新群（50-80記事） | 個別は低 | 14日 / 28日 | 28日 | 群全体で評価 |

> **統合判断**: Phase 1 では現行の14日間を維持（即時稼働を優先）。Phase 2 以降で28日間への拡張を推奨。

### 2.4 対照群（Control）の設計

**なぜ必要か**: Before/After比較だけでは以下を分離できない:
- Google アルゴリズム変更
- 季節需要の変動
- AI Overview 展開率の変化
- 競合の動き

**DID（差分の差分）計算:**
```
真の効果 = (処理群のAfter - Before) - (対照群のAfter - Before)
```

**対照群候補（data-measurement-expert 提案）:**

| 変更対象 | 対照群候補 | 選定理由 |
|---------|-----------|---------|
| /campus/10673 (MBTI一覧) | /campus/5196 (性格一覧) | 同カテゴリ、類似規模 |
| /campus/5504 (リクルートスーツ) | /campus/22136 (履歴書封筒) | 就活準備、Top3 |

**GAS への実装方法:**
- Config.gs に `CONTROL_GROUPS` マッピングを追加
- Main.gs で対照群URLに対しても fetchGA4Data() / fetchGSCData() を実行
- periodType: 'Before-Control' / 'After-Control' で記録

### 2.5 バイアス排除

| バイアス | リスク | 対処 |
|---------|-------|------|
| セレクションバイアス | 改善見込み記事のみ選定 | 対照群比較（DID）、選定基準を明記 |
| タイミングバイアス | 就活季節性との混同 | DID（季節要因は差分で相殺）、前年同月比併記 |
| 観測者バイアス | 担当者の確証バイアス | 閾値事前設定（+-5%）、自動判定の機械的適用 |
| サバイバーシップバイアス | 改善事例のみ報告 | 全変更の記録、月次サマリー、悪化アラート |
| 回帰効果 | 急落記事の自然回復を効果と誤認 | DID（対照群も同様なら回帰効果は相殺） |
| 複数施策の交絡 | 同時実施で効果帰属不明 | 段階的実施、施策タグ記録 |

### 2.6 A/Bテストの要否

**結論: 本ケースでは実施すべきでない**

理由:
1. SEO施策にA/Bテストは不適合（クローキングリスク）
2. サンプルサイズ不足（数週間〜数ヶ月必要）
3. 緊急性（リカバリー優先、機会損失回避）

代替手法として DID / 時系列介入分析 / 段階的ロールアウトを採用。

## 3. 統合アプローチ: マーケット分析フレームワーク

> 出典: market-analysis-framework.md

### 3.1 指標の因果連鎖

```
[変更実施]
    |
    v
【先行指標（1-7日）】
  - GSC表示回数
  - GSC平均掲載順位
    |
    v
【中間指標（7-28日）】
  - GSC CTR
  - GA4 オーガニックセッション
  - GA4 エンゲージメント率
    |
    v
【成果指標（28-90日）】
  - GA4 CV数
  - Ahrefs 推定トラフィック
```

### 3.2 統合KPI体系（3提案のハーモナイズ）

各提案のKPI重みに以下の差異があった:

| 指標 | efficiency/data提案 | market提案 |
|------|-------------------|------------|
| セッション数 | 40% | 25% |
| CTR | 30% | 30% |
| エンゲージメント率 | 20% | 15% |
| 直帰率（逆） | 10% | -- |
| 平均掲載順位 | -- | 20% |
| エンゲージメント時間 | -- | 10% |

**統合判断（シニアコンサルタント推奨）:**

Phase 1 では既存GASコード（Utils.gs）に実装済みの重みをそのまま使用する:

| KPI | 重み | 根拠 |
|-----|------|------|
| **オーガニックセッション数** | 40% | 実ビジネスインパクトの直接指標 |
| **GSC CTR** | 30% | AI Overview環境下で最重要。タイトル・バナー変更の直接効果 |
| **エンゲージメント率** | 20% | コンテンツ内部改善の効果 |
| **直帰率（逆指標）** | 10% | UX品質の補助指標 |

Phase 2 以降で、market-analyst 提案の**順位（20%）**と**エンゲージメント時間（10%）**を追加し、6指標体系に拡張する。

**判定基準:**

| 判定 | 条件 |
|------|------|
| **改善** | 重み付きスコア >= +5% |
| **悪化** | 重み付きスコア <= -5% |
| **変化なし** | -5% < スコア < +5% |

**信頼度レベル（Phase 2 以降で付記）:**

| 信頼度 | 条件 |
|--------|------|
| 高 | p < 0.01 かつ効果量 >= 最小値の1.5倍 |
| 中 | p < 0.05 かつ効果量 >= 最小値 |
| 低 | p < 0.05 だが効果量 < 最小値 |
| 判定保留 | p >= 0.05 |

### 3.3 統計検定手法（Phase 2 以降）

| 評価対象 | 手法 | 適用場面 |
|---------|------|---------|
| 記事単体 | Welch t検定 | 日次28日間のBefore/After比較 |
| CTR評価 | 二項比例検定 | 表示回数ベースのCTR比較 |
| クラスター | 混合効果モデル | 小規模記事群の一括評価 |
| DID分析 | 差分の差分 | 対照群との比較 |

**実務的に意味のある最小効果量:**

| 指標 | 最小効果量 |
|------|-----------|
| CTR | +1.5%pt以上 |
| セッション数 | +20%以上 |
| 順位 | +2位以上 |
| エンゲージメント率 | +3%pt以上 |
| エンゲージメント時間 | +15%以上 |

### 3.4 外部要因の排除

| # | 要因 | 影響の大きさ | 対処 |
|---|------|------------|------|
| 1 | 曜日効果 | 大（+-30%） | 曜日ダミー / 同曜日比較 |
| 2 | 就活季節性 | 大（年末→1月 +50%） | YoY比較 / 対照群 |
| 3 | Google アルゴリズム変更 | 特大（過去-65%） | DID / 競合ベンチマーク |
| 4 | AI Overview 展開率変化 | 大（CTR 1/3-1/5） | AI出現率記録 / 対照群 |
| 5 | 競合の動き | 中 | 競合順位取得 / DID |
| 6 | 祝日・長期休暇 | 中（-40%） | 祝日フラグ / 除外 |
| 7 | 他施策の同時実施 | 中〜大 | 段階的実施 / タグ記録 |

### 3.5 セグメント分析（Phase 2 以降）

| セグメント | 重要度 | 理由 |
|-----------|-------|------|
| モバイル vs デスクトップ | 高 | jokatsu訪問の70-80%がモバイル |
| 新規 vs リピーター | 高 | CTR改善は主に新規ユーザーに影響 |
| Organic Search | 最高 | 全トラフィックの91%を占有 |
| クエリタイプ別 | 中 | ブランド/一般/ロングテールの帰属分析 |

### 3.6 多重比較補正

| 同時評価記事数 | 補正方法 | 有意水準 |
|-------------|---------|---------|
| 1-5記事 | 補正なし | alpha = 0.05 |
| 6-20記事 | Benjamini-Hochberg（FDR） | FDR = 0.05 |
| 20記事超 | BH + 効果量フィルタ | FDR = 0.05 + 効果量閾値 |

## 8. 計測の限界と注意事項

### 8.1 因果推論の限界
- 未観測交絡因子（対象記事だけに特異的な要因）は排除不能
- SUTVA違反（内部リンクによる波及効果）の可能性
- 外的妥当性の限界（jo-katsu.com 固有の結果であり一般化には注意）

### 8.2 統計的有意性と実務的重要性の区別
- p < 0.05 でも効果が微小なら実務的には無意味
- 必ず効果量（セッション+20%、CTR+1.5pt 等）を併記

### 8.3 GA4サンプリングの問題
- 高トラフィックページで API が自動サンプリングを適用する場合がある
- レポートに samplingMetadatas の有無を確認・警告を付記
