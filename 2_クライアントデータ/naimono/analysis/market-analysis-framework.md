# クリエイティブ変更の効果測定フレームワーク

**対象サイト**: jo-katsu.com
**作成日**: 2026年2月17日
**データソース**: GA4 (356839446) / Ahrefs / GSC
**目的**: リライト・クリエイティブ変更の効果を統計的に正確に測定

## エグゼクティブサマリー

本フレームワークは、jo-katsu.comにおけるクリエイティブ変更の効果を**統計的有意性**を担保しながら測定するための、指標体系・分析手法・判定基準の総合設計書です。

**3つの核心**:
1. **階層化KPI**: Tier 1/2/3の指標体系（CTR 30%、セッション 25%等）
2. **統計的有意性**: Welch t検定、二項比例検定、混合効果モデル
3. **外部要因の排除**: DID分析、曜日補正、季節性考慮

**最重要推奨**:
- **CTR（30%重み）**: AI Overview環境下で生死を分ける指標
- **DID分析**: 対照群との比較で外部要因を相殺
- **最小効果量**: p値だけでなく実務的意味のある効果量を報告

## 2. 統計的に有意な効果測定の方法

### 2.1 基本設計: BA-C（Before-After with Control）

```
【処理群】変更した記事
  Before期間 → After期間
  → 差分を計算

【対照群】変更していない類似記事
  同一Before期間 → 同一After期間
  → 差分を計算

【DID効果】= 処理群の差分 - 対照群の差分
```

### 2.2 必要なサンプルサイズ

#### CTRの変化検出

| パラメータ | 値 |
|-----------|-----|
| 現在CTR（AI Overview下） | 5% |
| 検出したい最小効果量 | 2%pt (5%→7%) |
| 有意水準 | 0.05 |
| 検出力 | 0.80 |
| **必要表示回数** | **約2,400回** |

**jokatsu.comへの適用**:
- MBTI一覧（Vol 296k、8位）: 日次1,000回 → **3日**で十分
- 中規模KW（Vol 5k-10k、1-3位）: 日次100-300回 → **8-24日**
- 小規模KW（Vol 500-2k）: 日次20-50回 → **48-120日**（クラスター評価必須）

#### セッション数の変化検出

| パラメータ | 値 |
|-----------|-----|
| 平均日次セッション | ~30/日 |
| 標準偏差 | 平均の40-60% |
| 検出したい最小効果量 | 30%の相対変化 |
| **必要観測日数** | **28日以上** |

**重要**: 日次50未満のページは個別評価困難 → **クラスター単位**で評価

### 2.3 推奨計測期間

| 記事カテゴリ | 日次セッション | Before | After | 最短判定 |
|------------|--------------|--------|-------|---------|
| 大規模（MBTI一覧等） | 50+/日 | 28日 | 28日 | 14日で速報 |
| 中規模（各施策2記事） | 15-50/日 | 28日 | 42日 | 28日で速報 |
| 小規模（Vol 500-2k） | 5-15/日 | 28日 | 56日 | クラスター評価 |
| 鮮度更新群（50-80記事） | 個別は低 | 28日 | 28日 | 群全体で評価 |

### 2.4 統計検定の方法

#### 記事単体: Welch t検定

```
H0: 変更前後のセッション数の日次平均に差がない
H1: 変更前後のセッション数の日次平均に差がある
alpha = 0.05

手順:
1. Before/After 各28日間の日次セッション集計
2. 曜日補正（全サイト平均で正規化）
3. Welch t検定実施
4. p < 0.05 で有意
```

#### CTR評価: 二項比例検定

```
H0: 変更前後のCTRに差がない
H1: 変更前後のCTRに差がある

Z統計量 = (CTR_after - CTR_before) / 
          sqrt(CTR_pooled * (1-CTR_pooled) * (1/n_before + 1/n_after))
p < 0.05 で有意
```

#### クラスター評価: 混合効果モデル

```
Y_it = β0 + β1*After_t + β2*Treatment_i + β3*(After_t × Treatment_i) + u_i + e_it

β3 = DID推定量（クリエイティブ変更の因果効果）
β3 が有意（p < 0.05）→ 効果あり
```

### 2.5 効果量（Effect Size）の報告

**p値だけでなく効果量を必ず報告**

| 指標 | 実務的に意味のある最小効果量 |
|------|---------------------------|
| CTR | +1.5%pt以上 |
| セッション数 | +20%以上 |
| 順位 | +2位以上 |
| エンゲージメント率 | +3%pt以上 |
| エンゲージメント時間 | +15%以上 |

### 2.6 多重比較補正

| 同時評価数 | 補正方法 | 有意水準 |
|-----------|---------|---------|
| 1-5記事 | 補正なし | α = 0.05 |
| 6-20記事 | Benjamini-Hochberg（FDR） | FDR = 0.05 |
| 20記事超 | BH + 効果量フィルタ | FDR = 0.05 + 効果量閾値 |

## 3. 外部要因を排除するための手法

### 3.1 外部要因リスト

| # | 要因 | 影響 | 大きさ | 対処 |
|---|------|------|--------|------|
| 1 | **曜日効果** | セッション | 大（±30%） | 曜日ダミー/同曜日比較 |
| 2 | **就活季節性** | 全指標 | 大（年末→1月+50%） | YoY比較/対照群 |
| 3 | **Googleアルゴリズム** | 順位 | 特大（過去-65%） | DID/競合ベンチマーク |
| 4 | **AI Overview展開** | CTR | 大（1/3-1/5） | AI出現率記録/対照群 |
| 5 | **競合の動き** | 順位 | 中 | 競合順位取得/DID |
| 6 | **祝日・長期休暇** | セッション | 中（-40%） | 祝日フラグ/除外 |
| 7 | **検索Vol変動** | 表示回数 | 中 | GSC表示で正規化 |
| 8 | **他施策の同時実施** | 全指標 | 中〜大 | 段階的実施/タグ記録 |

### 3.2 具体的対処方法

#### 曜日効果
```
正規化セッション数 = 実測セッション / 同曜日の全サイト平均セッション
```

#### 就活季節性
- DID分析（対照群も同じ季節影響）
- 前年同月比併記
- 計測期間の明示（「3月計測は季節需要含む」等）

#### Googleアルゴリズム変更
- DID分析（対照群も同様に影響）
- 競合ベンチマーク（同KWの他サイト順位）
- アップデート日時の記録

#### AI Overview展開率の変化
- AI出現率の手動記録
- 対照群比較（CTR減少が全体トレンドか個別記事か）

## 4. セグメント分析の推奨事項

### 4.1 デバイス別分析

| セグメント | 重要度 | 理由 |
|-----------|-------|------|
| モバイル | 高 | jokatsu訪問の70-80%を占める |
| デスクトップ | 中 | 詳細情報閲覧層 |
| タブレット | 低 | 数%のみ |

**推奨**: モバイルを主軸、デスクトップは補助

### 4.2 流入元別分析

| セグメント | 重要度 | 評価指標 |
|-----------|-------|---------|
| Organic Search | 最高 | 全Tier 1指標 |
| Direct | 中 | ブランド力の評価 |
| Social | 低 | 一時的流入 |
| Referral | 低 | 被リンク効果 |

**推奨**: Organic Searchに特化（既存実装済）

### 4.3 新規/リピーター別分析

| セグメント | 重要度 | 評価指標 |
|-----------|-------|---------|
| 新規ユーザー | 高 | CTR改善の主ターゲット |
| リピーター | 中 | コンテンツ品質の指標 |

**推奨**: 新規ユーザーのCTR・エンゲージメントを主評価

### 4.4 クエリタイプ別分析（GSC）

| タイプ | 評価指標 | 分析目的 |
|--------|---------|---------|
| ブランドKW | 表示・CTR | ブランド認知 |
| 一般KW | 順位・CTR・表示 | SEO効果の主軸 |
| ロングテール | キーワード数 | 網羅性の評価 |

**推奨**: 一般KWを主軸、ロングテールは補助

## 5. 判定基準

### 5.1 重み付きスコア計算

```
重み付きスコア = 
  (セッション変化率 × 0.40) +
  (CTR変化率 × 0.30) +
  (エンゲージメント率変化 × 0.20) +
  (直帰率変化（逆） × 0.10)
```

### 5.2 判定閾値

| 判定 | 条件 | 閾値 |
|------|------|------|
| **改善** | 重み付きスコア >= +5% | `VERDICT_IMPROVED_THRESHOLD: 0.05` |
| **悪化** | 重み付きスコア <= -5% | `VERDICT_DEGRADED_THRESHOLD: -0.05` |
| **変化なし** | -5% < スコア < +5% | - |

### 5.3 信頼度レベルの付記（推奨）

| 信頼度 | 条件 |
|--------|------|
| 高 | p < 0.01 かつ 効果量 >= 最小値の1.5倍 |
| 中 | p < 0.05 かつ 効果量 >= 最小値 |
| 低 | p < 0.05 だが効果量 < 最小値 |
| 判定保留 | p >= 0.05 |

### 5.4 判定時の追加考慮事項

- **対照群の変動**: 対照群も同方向に変化 → サイト全体のトレンド
- **季節イベント**: 計測期間中の就活イベント有無
- **GSCデータ欠損**: GSC権限なし → GA4のみで判定
- **サンプルサイズ**: 低トラフィックページ → 「要継続観察」

## 7. 重要な注意事項

### 7.1 統計的有意性 ≠ 実務的重要性

- p < 0.05 でも効果が微小なら実務的には無意味
- **必ず効果量を報告**

### 7.2 因果推論の限界

- 準実験（RCTではない）
- 未観測交絡因子の可能性
- 一般化の限界

### 7.3 GSC権限の最優先取得

GSCデータなしでは：
- CTR比較不可
- 順位追跡不可
- クエリ分析不可

→ **最優先で権限取得**
