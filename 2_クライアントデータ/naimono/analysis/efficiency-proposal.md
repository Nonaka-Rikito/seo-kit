**作成日**: 2026年2月17日
**対象**: jokatsu.com バナー/CTA クリエイティブの差替え効果測定
**目的**: 正確なデータ計測を最小限の手動作業で継続運用する体制の構築

## 1. 現状分析

### 1.2 GAS システムの設計概要（既存）

既存の GAS コードは以下の機能を実装済み。

```
[Google フォーム] ─送信→ [onFormSubmit()]
                            │
                            ├─ Before データ取得（GA4 + GSC）
                            ├─ スプレッドシートに記録
                            ├─ Slack 通知「計測開始」
                            └─ 17日後の時限トリガー設定
                                      │
                                      ▼
                              [generateReport()]
                                      │
                                      ├─ After データ取得（GA4 + GSC）
                                      ├─ Before/After 比較・判定
                                      ├─ スプレッドシートに記録
                                      └─ Slack 通知「レポート」
```

**計測指標（全16項目）**:
- GA4: セッション数, PV数, エンゲージセッション数, エンゲージメント率, 平均セッション時間, 直帰率, CV数
- GA4 (スクロール): 25% / 50% / 75% / 90% 到達数
- GSC: クリック数, 表示回数, CTR, 平均掲載順位
- GSC (画像検索): 画像クリック数, 画像表示回数

### 1.3 現状の課題

| # | 課題 | 影響度 |
|---|------|--------|
| 1 | GAS システムが未デプロイ（Config.gs がプレースホルダのまま） | **致命的** |

| 5 | ダッシュボードシートが未作成 | 中 |

## 2. 推奨する運用フロー

### 2.1 Phase 1: 最速導入（即日〜3日）

**原則: 既存 GAS コードをそのまま活用し、最小限の設定だけで稼働させる。**

```
┌─────────────────────────────────────────────────────────────┐
│                    運用フロー（Phase 1）                      │
│                                                             │
│  【手動】                  【自動】                          │
│                                                             │
│  ① バナー差替え            ③ Before データ自動取得            │
│     （通常業務）               GA4: 過去14日分                │
│         │                      GSC: 過去14日分               │
│         ▼                                                   │
│  ② Google フォームに        ④ スプレッドシートに自動記録       │
│     記事URLを入力                                            │
│    （所要時間: 30秒）       ⑤ Slack に「計測開始」通知         │
│                                                             │
│         ～～～ 17日経過 ～～～                                │
│                                                             │
│                            ⑥ After データ自動取得             │
│                            ⑦ Before/After 比較・判定          │
│                            ⑧ Slack にレポート送信             │
│                            ⑨ スプレッドシートに結果記録        │
│                                                             │
│  人が行う作業: ②のみ（30秒/件）                              │
│  それ以外は全自動                                            │
└─────────────────────────────────────────────────────────────┘
```

#### 担当者と作業分担

| 役割 | 担当者 | 作業内容 | 頻度 |
|------|--------|----------|------|
| **バナー差替え実行者** | コンテンツ担当 | CMS でバナー画像を変更後、Google フォームに URL を入力 | 差替え発生時（月2〜5回想定） |
| **初期設定** | 技術担当（1回のみ） | Config.gs の設定値入力、GAS デプロイ、Slack Webhook 設定 | 初回のみ |
| **レポート確認** | マネージャー | Slack 通知を確認し、意思決定 | 月2〜5回 |
| **定期監視** | 誰でも | スプレッドシートで蓄積データを確認 | 月1回 |
