## 3. 機能別システムフロー

### 3-A. 画像生成AI

```
ユーザー入力（日本語プロンプト + エフェクト + 人物設定 + アスペクト比）
    ↓
POST /v1/ai/generate/image
    ↓ バックエンド処理
    ├─ 日本語→英語 自動翻訳（Gemini Flash推定）
    ├─ ネガティブ表現を自動分離 → negative prompt化
    ├─ エフェクト・人物設定をシステムプロンプトとして付加
    └─ Google系画像生成モデルに4枚同時生成を依頼
    ↓
GET /v1/ai/request?processing_ids[]=xxxxx（約4秒間隔ポーリング）
    ↓ 生成完了（約10秒）
S3にアップロード → URLを返却 → フロントで表示
```

**参考画像からの生成:**
1. `POST /v1/ai/upload` で画像アップロード
2. `POST /v1/ai/generate/prompt` でマルチモーダルLLMが画像解析→プロンプト自動生成
3. 生成プロンプトで上記フローを実行

### 3-B. バナー生成AI（最も複雑）

```
Step 1: LP解析
    POST /v1/ai/banner/lp（URL入力）
    ↓ バックエンドがLPをクロール
    ├─ タイトル、OGタイトル、OG説明を抽出
    ├─ H1/H2見出し、テキストを抽出
    └─ 画像素材を抽出・一覧化
    ※ JavaScriptの動的読み込みは非対応

Step 2: ユーザー設定
    ├─ 使用画像を選択（最大10枚）
    ├─ プラットフォーム選択（10種）
    ├─ ターゲットセグメント（9セグマップ）
    ├─ 訴求軸（8種類）
    ├─ テキスト量（5段階）
    └─ カスタム指示（任意）

Step 3: バナー生成
    POST /v1/ai/banner
    ↓ バックエンド処理パイプライン
    ├─ LP解析結果 + ターゲット + 訴求軸 → Gemini Proでコピーテキスト生成
    ├─ デザイン構成決定（テンプレート + AI判断）
    ├─ 画像素材の生成/加工
    └─ テキスト配置・レイアウト処理（Proモードは日本語テキスト入れ対応）
    ↓
8枚 or 16枚を一括生成 → ポーリングで進捗監視 → S3から配信
```

**優先順位ロジック:**
カスタム指示 > テキスト量 > ターゲット・訴求軸 > テンプレート

**プラットフォーム選択肢（実UI確認、10種）:**
Instagram / Facebook / TikTok / LINE / YouTube / SmartNews / Yahoo / X / Google(α版) / Pangle

**ターゲット:** 9セグマップ（9項目、実UI確認済み）
**訴求軸（8種類、実UI確認済み）:** トレンド / 限定 / ベネフィット / 逃避 / 価格 / ネガティブ / 権威性 / 品質・機能

### 3-C. 画像編集AI

- `POST /v1/ai/edit/image` — 元画像 + 編集プロンプトで修正
- **Fastモード**: 1cr/枚
- **Proモード**: 3cr/枚、2K出力、日本語テキスト対応
- 最大2枚の参考画像を追加可能
- 編集履歴機能でバージョン管理

### 3-D. 動画生成AI

- **Veo 3.1**（Google DeepMind）
- 長さ: 4秒 / 6秒 / 8秒の3段階
- リップシンク音声生成対応（発話内容を引用符で指定）
- キーフレーム指定機能あり
- `POST /v1/ai/generate/video`
- 生成時間: 約45秒〜1分

## 4. プロンプトエンジニアリング（ヘルプセンターで確認済み）

| 処理 | 説明 |
|------|------|
| 日本語→英語 自動翻訳 | プロンプトを自動で英訳してモデルに渡す |

| 発話部分の翻訳除外 | 引用符内のテキストはそのまま保持（リップシンク用） |
| エフェクト・人物プリセット | 選択に応じてシステムプロンプトとして付加 |

## 6. 内製化に必要なコンポーネント

### AI基盤層
- Google Cloud AI / Vertex AI のAPIアクセス（Gemini Pro/Flash + Veo 3.1 + 画像生成モデル）
- 必要に応じてOpenAI APIも補助的に利用

### プロンプトエンジニアリング層
- 日本語→英語の自動翻訳パイプライン
- ネガティブ表現の自動分離処理
- エフェクト・人物設定のプリセット管理
- 発話部分の引用符検出と翻訳除外ロジック
- LP解析結果→マーケティングコピー生成のプロンプトテンプレート

### LP解析パイプライン（バナー生成用）
- Webスクレイパー（Puppeteer/Playwright等）でLPをクロール
- テキスト抽出（タイトル、OG情報、見出し、本文）
- 画像抽出とリスト化
- 9セグマップ × 8訴求軸のマトリクスに基づくコピー生成

### インフラ層
- 非同期ジョブキュー（生成リクエスト→バックグラウンド処理→完了通知）
- ポーリング or WebSocket による進捗通知
- オブジェクトストレージ（S3等、サムネイル自動生成含む）
- クレジット/課金管理システム

### フロントエンド
- React系SPAのUI
- 画像D&D、プロンプトサンプル機能、編集履歴機能
